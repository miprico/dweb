Flexbox: distribuir y alinear con intención

Flexbox es el sistema de CSS que permite distribuir elementos en un contenedor de forma flexible y controlada, en filas o columnas. Aprenderás display: flex, justify-content, align-items, flex-direction y flex-wrap. Y a través de eso, a pensar en cómo distribuimos la atención, el reconocimiento y el espacio en un grupo.

🍕Con compañeros · Repartir no es repartir igual

En el trabajo de grupo de Biología, son cuatro. La docente dice: "distribúyanse las partes". En cinco minutos, Paula y Ernesto ya tienen las dos partes más visibles — la introducción y las conclusiones. A Sofía le toca la metodología. A Jair le toca las referencias. El producto final parece de dos personas. Distribuir no es lo mismo que distribuir equitativamente. En Flexbox, justify-content decide cómo se reparte el espacio disponible entre los elementos. Y no todos los valores reparten igual.

💭 ¿En los trabajos grupales de tu colegio, cómo se distribuye realmente el trabajo? ¿Hay alguien que siempre hace más — o menos?
📏En casa · Alinearse no es volverse igual

Los tres hermanos de la familia Pedraza son completamente distintos: Natán estudia ingeniería, Camila quiere ser bailarina, el pequeño Diego apenas está descubriendo qué le gusta. La mamá los trata diferente — no por preferencia, sino porque alinearse a cada uno requiere cosas distintas. Alinearlos a todos por el mismo eje no significa que deban ser iguales. En Flexbox, align-items controla cómo se alinean los elementos en el eje cruzado — y hay varias formas de hacerlo.

💭 ¿Tu familia te trata de acuerdo a quién eres o te "alinea" a un estándar que no siempre te representa?
🔄En el colegio · Cuando no cabe todo en una fila

En el evento de la ENSDB, todos los grupos querían estar en la primera fila del auditorio. No cabían. Un docente propuso: "si no caben en una fila, se forma una segunda". Algunos grupos aceptaron, otros se molestaron porque "segunda fila" les parecía menos importante. flex-wrap hace exactamente eso: cuando los elementos no caben en una línea, se permite que pasen a la siguiente. El problema no es la segunda fila — es que creemos que vale menos.

💭 ¿Hay situaciones en tu vida donde "pasar a la segunda fila" sería la solución práctica, pero el ego no lo permite?

El trabajo de Biología con distribución desigual. Los hermanos Pedraza que necesitan alinearse diferente. Los grupos del evento que no caben en una fila. Flexbox es el sistema CSS que resuelve estos problemas de distribución y alineación — con control total sobre quién va dónde, con cuánto espacio, y en qué dirección.

01

display: flex — el contenedor y sus hijos

flex-direction · flex-wrap · gap — las bases que controlan el flujo

Explicación

Cuando aplicas display: flex a un elemento, ese elemento se convierte en el contenedor flex y sus hijos directos se convierten en items flex. El contenedor controla cómo se distribuyen y alinean esos hijos usando el eje principal y el eje cruzado.

Los dos ejes del contenedor flex:
flex-base.css
.contenedor { display: flex; /* activa flexbox */ /* flex-direction: la dirección del eje principal */ flex-direction: row; /* → izq a der (por defecto) */ /* flex-direction: row-reverse; ← der a izq */ /* flex-direction: column; ↓ de arriba hacia abajo */ /* flex-direction: column-reverse; ↑ de abajo hacia arriba */ /* flex-wrap: qué pasa cuando no caben en una línea */ flex-wrap: nowrap; /* todos en una línea, se comprimen (por defecto) */ /* flex-wrap: wrap; pasan a la línea siguiente */ /* flex-wrap: wrap-reverse; segunda línea arriba */ /* gap: espacio entre items */ gap: 16px; /* igual en filas y columnas */ /* gap: 12px 24px; row-gap column-gap */ }

02

justify-content y align-items: distribuir y alinear

Los dos ejes, los dos controles — cómo se reparte el espacio y dónde se ubica cada quien

Explicación

justify-content distribuye el espacio en el eje principal (por defecto: horizontal). align-items alinea los items en el eje cruzado (por defecto: vertical). Juntas, estas dos propiedades permiten centrar perfectamente un elemento, distribuir espacio equitativamente, o alinear todo al inicio o al final.

justify-align.css
/* El truco más usado en CSS: centrar un elemento en su contenedor */ .centrado-perfecto { display: flex; justify-content: center; /* centro horizontal */ align-items: center; /* centro vertical */ min-height: 100vh; /* ocupa toda la pantalla */ } /* Nav con logo a la izquierda y links a la derecha */ .nav { display: flex; justify-content: space-between; align-items: center; padding: 16px 32px; } /* Tarjetas en fila que se rompen si no caben */ .galeria { display: flex; flex-wrap: wrap; gap: 20px; justify-content: flex-start; } /* align-self: un item individual se alinea diferente a los demás */ .item-especial { align-self: flex-end; /* solo este item se va al fondo */ }

La metáfora del trabajo grupal: justify-content: space-between pone a Paula y Ernesto en los extremos visibles. space-evenly hubiera distribuido el protagonismo de manera más justa. El valor de justify-content que eliges para tu grupo refleja cómo entiende la distribución del reconocimiento.

03

flex-grow, flex-shrink, flex-basis y order

Cuánto crece, cuánto se encoge y en qué orden — propiedades de los items flex

Explicación

Estas propiedades se aplican a los items flex (los hijos del contenedor), no al contenedor. Controlan cómo cada item individual crece para ocupar espacio disponible, se encoge si falta espacio, cuál es su tamaño base, y en qué orden aparece.

flex-items.css
/* flex-grow: cuánto crece un item para ocupar espacio disponible */ /* 0 = no crece. 1 = crece igual que todos. 2 = crece el doble */ .item-principal { flex-grow: 2; } /* ocupa el doble que los demás */ .item-normal { flex-grow: 1; } /* crece igual que sus hermanos */ .item-fijo { flex-grow: 0; } /* no crece — mantiene su tamaño */ /* flex-shrink: cuánto se encoge si falta espacio */ /* 1 = se encoge (por defecto). 0 = no se encoge aunque no quepa */ .item-rigido { flex-shrink: 0; } /* flex-basis: tamaño base antes de aplicar grow/shrink */ .sidebar { flex-basis: 260px; } /* empieza con 260px */ .main-content { flex-basis: 0; } /* empieza desde 0, crece con flex-grow */ /* SHORTHAND flex: grow shrink basis */ .flex-uno { flex: 1; } /* flex: 1 1 0 — crece igual, se encoge, desde 0 */ .flex-auto { flex: auto; } /* flex: 1 1 auto — el más común */ .flex-none { flex: none; } /* flex: 0 0 auto — no crece ni se encoge */ /* order: cambia el orden visual sin cambiar el HTML */ .primero { order: -1; } /* aparece antes que todos */ .ultimo { order: 99; } /* aparece al final */

A1

El nav y el header de mi proyecto con Flexbox

Individual · 25 minutos · Autoconciencia — elegir cómo distribuyo lo que muestro de mí
⏱ 25 min 👤 Individual 💻 CSS externo 🔍 SEL: Autoconciencia
🎯La pregunta de diseño

Antes de escribir Flexbox, piensa: ¿cómo distribuyes la información en tu header? ¿Tu nombre va a la izquierda y la nav a la derecha? ¿Todo centrado? ¿El logo pequeño arriba y la descripción grande debajo? Cada valor de justify-content y align-items que elijas dice algo sobre cómo quieres presentarte.

💭 ¿Cómo quieres que se distribuya la información más importante de tu página — y qué tan centrado quieres que se vea?
  • 1

    Convierte el <nav> de tu proyecto en un contenedor flex

    Con display: flex, justify-content: space-between para logo a la izquierda y links a la derecha, y align-items: center para alineación vertical. El gap entre los links del nav también con Flexbox.

  • 2

    Convierte el <header> de tu index.html en flex column

    Con flex-direction: column, align-items: center (todo centrado) y gap entre el h1 y el párrafo descriptivo. Si hay imagen o avatar, que sea un flex item más.

  • 3

    Crea una galería de tarjetas con flex-wrap

    En tu mi_historia.html, convierte la sección de habilidades o intereses en un contenedor flex con flex-wrap: wrap y gap: 16px. Cada tarjeta con flex: 0 0 calc(33% - 12px) en pantallas grandes.

  • 4

    Aplica flex-grow para crear un layout de dos columnas

    En alguna sección de tu proyecto: un elemento con flex: 1 (contenido principal) y otro con flex: 0 0 260px (aside o barra lateral).

📋 Tu CSS debe incluir
  • Nav con display: flex + justify-content: space-between + align-items: center
  • Header con flex-direction: column + align-items: center + gap
  • Al menos una galería con flex-wrap: wrap
  • Al menos un layout con flex-grow o flex: 1
  • Uso de justify-content con al menos dos valores diferentes
  • Comentarios CSS explicando por qué elegiste cada valor de justify-content y align-items
🔍 SEL · Autoconciencia

Cómo distribuyes el espacio en tu página refleja cómo organizas lo que quieres mostrar de ti

Elegir justify-content: center o space-between es una decisión estética, pero también una decisión de énfasis: ¿qué quieres que esté en el centro? ¿Qué quieres que ocupe los extremos visibles? ¿Cuánto espacio le das a cada parte de tu historia? En tu bitácora:

  • → ¿Qué valor de justify-content elegiste para tu header? ¿Qué dice eso de cómo quieres presentarte?
  • → ¿Usaste flex-grow para dar más espacio a algo específico? ¿Por qué esa sección merece más espacio que las demás?
  • → ¿Hay partes de tu historia personal que "crecen" naturalmente cuando hablas de ellas, como un item con flex-grow: 2?

¿Qué justify-content usamos en la vida?

Parejas · 20 minutos · Conciencia social — analizar cómo distribuimos atención y reconocimiento
⏱ 20 min 👥 Parejas 💻 + conversación 🌍 SEL: Conciencia Social
💡Volvamos al trabajo de Biología — ¿qué justify-content era ese?

Paula y Ernesto tomaron las partes visibles. Sofía y Jair tomaron las invisibles. Ese grupo usó justify-content: space-between — los reconocidos en los extremos, los demás en el medio sin protagonismo. ¿Cómo hubiera sido ese trabajo con space-evenly? ¿Qué cambiaría en el resultado — y en las relaciones?

💭 ¿Qué valor de justify-content describes mejor cómo se distribuye el reconocimiento en tu salón de clase?
  • 1

    Construyan juntos una página HTML: "Nuestro grupo de trabajo" (8 min)

    Con 4 tarjetas flex — una por cada "integrante hipotético" de un grupo. Cada tarjeta tiene: nombre, parte del trabajo asignada, y un número de flex-grow que refleja cuánto protagonismo tiene esa persona en el trabajo real.

  • 2

    Implementen dos versiones del mismo grupo (7 min)

    Versión A: flex-grow desigual — alguien tiene 3, alguien tiene 1, alguien tiene 0. Versión B: todos con flex: 1 — mismo espacio para todos. Comparen cómo se ve cada versión.

  • 3

    Escriban un análisis en el HTML (5 min)

    Un <p> por versión explicando: ¿qué tipo de grupo representa cada versión? ¿Cuál se parece más a los grupos reales de tu experiencia? ¿Cuál sería más justo?

📋 Producto esperado A2
  • Página con dos versiones del mismo grupo en flex
  • flex-grow diferenciado en versión A, flex: 1 en versión B
  • Análisis en HTML conectando cada versión con dinámicas grupales reales
🌍 SEL · Conciencia Social

justify-content: space-evenly es el ideal igualitario — pero los grupos reales raramente lo aplican

La conciencia social incluye "reconocer las fortalezas de los demás". En la mayoría de grupos escolares, el reconocimiento no se distribuye con space-evenly — hay quienes siempre tienen flex-grow: 3 y quienes tienen 0. El código puede mostrar ese patrón con más claridad que cualquier discurso. En tu bitácora:

  • → ¿Qué flex-grow te asignarías en los grupos de trabajo de tu salón? ¿Es el que quisieras?
  • → ¿Hay alguien en tu salón que siempre tiene flex-grow: 0 en los grupos — que nunca ocupa espacio de protagonismo? ¿Qué podrías hacer?
  • → Si tu familia fuera un contenedor flex, ¿qué justify-content la describe mejor? ¿Es el que te gustaría?
A3

El layout del salón: una página completa con Flexbox

Grupos de 4 · 20 minutos · Habilidades relacionales — construir una estructura que nos muestre a todos
⏱ 20 min 👥 Grupos de 4 💻 Archivo colectivo 💬 SEL: Habilidades Relacionales
🏫El evento de la ENSDB — wrap o no wrap

Los grupos del evento no cabían en una fila. La solución era flex-wrap. El desafío no era técnico — era de ego. Esta actividad propone que el grupo construya una página con una galería de tarjetas de todos sus integrantes, usando Flexbox para que todos quepan y todos tengan el mismo espacio — independientemente del tamaño de la pantalla.

💭 ¿Cómo asegura el grupo que todos tienen espacio visible — sin que nadie quede en una "segunda fila" ignorada?
  • 1

    Construyan equipo.html — una tarjeta por integrante (8 min)

    Cada tarjeta incluye: nombre, una frase que los representa, y un emoji o color que los identifica. El CSS debe usar flex-wrap: wrap para que las tarjetas pasen a la siguiente fila si la pantalla es pequeña, y justify-content: center para que el grupo siempre se vea centrado.

  • 2

    Apliquen estas propiedades al contenedor de tarjetas

    equipo.css
    .galeria-equipo { display: flex; flex-wrap: wrap; gap: 20px; justify-content: center; } .tarjeta-integrante { flex: 0 0 calc(50% - 10px); /* 2 por fila */ /* o flex: 1 para que se adapten */ }
  • 3

    Discutan: ¿todos los integrantes tienen el mismo flex-grow?

    La discusión del grupo: ¿deberían tener flex: 1 todas las tarjetas (mismo tamaño) o flex-grow distinto (tamaños diferentes)? Implementen la decisión consensuada y documenten por qué en un comentario CSS.

📋 Producto esperado A3
  • Página equipo.html con galería flex de tarjetas personales
  • flex-wrap: wrap para que funcione en cualquier pantalla
  • justify-content consensuado y documentado
  • Un comentario CSS colectivo: ¿por qué eligieron ese valor de flex para las tarjetas?
💬 SEL · Habilidades Relacionales

Un grupo bien construido con Flexbox es aquel donde todos tienen espacio — no uno donde uno domina

Las habilidades relacionales incluyen "practicar el trabajo en equipo". La decisión sobre el flex-grow de cada tarjeta es una decisión sobre equidad en el grupo: ¿todos merecen el mismo espacio visual o hay jerarquías? ¿Qué mensaje envía la página sobre cómo se relacionan como equipo? En tu bitácora:

  • → ¿El grupo eligió flex: 1 (todos iguales) o flex distinto? ¿Fue fácil o difícil llegar a ese acuerdo?
  • → ¿La galería con flex-wrap da la misma visibilidad a todos? ¿O alguien termina en una segunda fila que se ve menos?
  • → ¿Hay algo que cambiarías de la distribución de tareas en tu grupo de clase después de trabajar con Flexbox hoy?

Criterios de valoración · Guía 14

¿Cómo se evaluará tu trabajo?

CriterioBásico (1–2)Satisfactorio (3)Superior (4–5)
Contenedor flex funcional
Técnico		 display: flex aplicado sin justify-content ni align-items. Usa display: flex con justify-content y align-items apropiados en al menos dos contenedores distintos. Cada contenedor flex tiene el combination correcta de direction, wrap, justify-content y align-items según su propósito de diseño.
flex-wrap y layouts responsivos
Técnico		 No usa flex-wrap — los items se comprimen o desbordan. Usa flex-wrap: wrap en la galería de tarjetas con gap adecuado. El layout funciona bien en pantallas anchas y estrechas gracias a flex-wrap y flex-basis calculado.
flex-grow y distribución
Técnico		 No usa flex-grow ni flex: 1. Usa flex: 1 en el contenido principal y flex: 0 0 Xpx en el aside. El uso de flex-grow refleja una decisión consciente sobre qué merece más espacio — con comentario que lo justifica.
Distribución justa en el grupo
SEL · Conciencia Social		 La galería del equipo usa flex-grow desigual sin reflexión. El grupo discutió y documentó su elección de flex-grow en comentario CSS. La reflexión sobre justify-content como metáfora de distribución de reconocimiento es genuina y específica — no genérica.
Conexión Flexbox - vida grupal
SEL		 La reflexión no conecta Flexbox con dinámicas grupales reales. Identifica qué valor de justify-content describe mejor un grupo real de su vida. La reflexión es precisa: nombra personas, situaciones específicas y propone algo concreto para cambiar hacia un "space-evenly" más real.
