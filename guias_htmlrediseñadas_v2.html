¿Qué es HTML? Estructura básica y comentarios

HTML estructura el contenido de la web. Hoy creas tu primer documento real: cinco líneas que el navegador entiende, y aprendes qué se ve y qué permanece oculto.

!DOCTYPE · html · head · body · meta · titlecomentarios <!-- -->Autoconciencia
1 / 10

Al terminar esta guía puedes

  • Crear un archivo .html válido con la estructura mínima de 5 partes
  • Explicar para qué sirve cada etiqueta del esqueleto
  • Documentar tu código con comentarios que el navegador ignora
🦴

El esqueleto que toda página necesita

Todo documento HTML empieza con la misma base. Sin esta estructura, el navegador no sabe qué hacer con el archivo. Los comentarios <!-- texto --> son invisibles para el usuario — solo los ve quien lee el código.

index.html
<!DOCTYPE html> ← versión de HTML <html lang="es"> ← raíz del documento <head> ← info INVISIBLE para el usuario <meta charset="UTF-8"> ← permite tildes y ñ <title>Mi primera página</title> </head> <body> ← todo lo VISIBLE va aquí <!-- Este comentario no se ve en el navegador --> Hola, soy mi primera página web. </body> </html>
Resultado en el navegador

Hola, soy mi primera página web.

Regla de oro: Cada etiqueta que abre <tag> debe cerrarse </tag>. Excepciones: <meta> y <!DOCTYPE> no se cierran.

Encabezados, párrafos y formato de texto

El texto no es todo igual. Los encabezados crean jerarquía. Los párrafos dan espacio. <strong>, <em> y <u> marcan énfasis. Conviertes tu index.html en una página con contenido estructurado real.

  • Escribir párrafos con <p> y aplicar énfasis con strong y em con intención
  • Completar el body de index.html con contenido real y estructurado
🔤

Jerarquía y énfasis: no todo puede ser h1

Los encabezados van de h1 (el más importante, uno solo por página) hasta h6. Los párrafos son <p>. El énfasis visual tiene tres grados: <strong> importancia real (negrita semántica), <em> énfasis de tono o voz (cursiva), <u> subrayado — raramente usado en web porque confunde con enlace.

Es que <strong>lo que me piden estudiar</strong> y <em>lo que quiero aprender</em> son cosas distintas. </p>

📋

Listas, espacio y la voz de otros

<ul> lista sin orden (puntos). <ol> lista numerada — el orden importa. <br> fuerza nueva línea sin párrafo nuevo. <hr> traza una línea divisoria horizontal. <blockquote> para citas largas (bloque aparte), <q> para citas cortas dentro del texto.

mi_historia.html → <body>

Resultado en el navegador

Lo que me mueve

  • Diseño gráfico — aunque nadie me lo enseñó
  • La música del barrio los domingos
"Uno no escoge de dónde viene, pero sí puede escoger a dónde va."
— Mi abuela
Diferencia clave: <ul> = el orden no importa (hobbies, ingredientes). <ol> = el orden sí importa (pasos de un proceso, ranking, prioridades).

Enlaces e imágenes: conectar y mostrar

Con <a> el proyecto pasa de archivos sueltos a un sitio navegable. Con <img> agregas imágenes. Al terminar tienes 3 páginas enlazadas y una imagen con alt descriptivo.

a · href · target="_blank" · img · src · alt · widthHabilidades Relacionales · Autoconciencia
4 / 10

Al terminar esta guía puedes

  • Crear enlaces internos (entre tus páginas) y externos (a otros sitios web)
  • Insertar una imagen con src, alt y width correctamente
  • Tener 3 páginas del proyecto navegables entre sí
🔗

El enlace y la imagen: los dos conectores de la web

El enlace <a href="..."> crea una puerta a otro lugar. Para páginas propias escribes solo el nombre del archivo. Para sitios externos escribes la URL completa y añades target="_blank" para abrir en pestaña nueva. La imagen <img> siempre necesita alt — describe la imagen para quien no puede verla.

index.html → dentro del <body>
<!-- Enlace interno: misma carpeta --> <a href="mi_historia.html">Mi historia</a> <!-- Enlace externo: URL completa + nueva pestaña --> <a href="https://code.org" target="_blank">Aprende a programar</a> <!-- Imagen con src, alt descriptivo y ancho fijo --> <img src="mi_foto.jpg" alt="Camila sonriendo en el puerto de Barranquilla" width="300" >
Si src es incorrecto, el navegador muestra el alt en su lugar. Por eso el alt siempre debe describir la imagen — no "foto", no "imagen", sino qué hay en ella.

Divisiones y estructura semántica

<div> agrupa sin significado. Los semánticos <header> <nav> <main> <section> <aside> <footer> le dan nombre a cada zona. Reestructuras las 3 páginas del proyecto sin cambiar su contenido visible.

  • Envolver el contenido de tus páginas en header, nav, main, aside y footer
  • Mejorar la estructura sin que nada visible cambie para el usuario
🧱

Cajas con nombre vs. cajas genéricas

<div> es una caja sin significado — agrupa, pero no dice qué hay adentro. Los elementos semánticos le dicen al navegador y a cualquier persona que lea el código qué tipo de contenido hay en cada zona. Solo hay un <main> por página. El resto puede repetirse.

Estructura de una tabla y colspan

Toda tabla tiene filas (<tr>), celdas de datos (<td>) y celdas de encabezado (<th>). Se organiza en <thead> (cabecera), <tbody> (datos) y <tfoot> (resumen). colspan="2" hace que una celda ocupe 2 columnas — útil para agrupar encabezados.

Un formulario completo: todos los tipos en uno

El <label> vinculado al input por for="id" es fundamental — sin él, al hacer clic en el texto del label el campo no se activa. El type del input define qué tipo de respuesta acepta. required hace el campo obligatorio — el formulario no envía si está vacío.

Video, audio y multimedia

Incrustar multimedia es técnicamente simple. Lo complejo es elegir qué mostrar — y cómo. <source> ofrece formatos alternativos para que funcione en cualquier navegador.

video · audio · source · controls · poster · autoplay · muted · loopAutoconciencia · Conciencia Social
8 / 10

Al terminar esta guía puedes

  • Incrustar video y audio con <video> y <audio> y sus atributos de control
  • Usar <source> para ofrecer el archivo en más de un formato
  • Elegir conscientemente qué mostrar y con qué controles
🎬

Video y audio: mostrar con control

<video> y <audio> tienen la misma lógica: contenedor + atributos de control + <source> dentro con el archivo. controls muestra la barra de reproducción — sin él el usuario no puede reproducir. poster es la imagen antes de reproducir. autoplay + muted reproduce automáticamente sin sonido (requerido por los navegadores modernos).

mi_historia.html → sección multimedia
<!-- VIDEO con controles e imagen previa --> <video controls poster="imagen_previa.jpg" width="400"> <source src="mi_video.mp4" type="video/mp4"> Tu navegador no soporta video. </video> <!-- AUDIO con dos formatos alternativos --> <audio controls> <source src="cancion.mp3" type="audio/mpeg"> <source src="cancion.ogg" type="audio/ogg"> </audio> <!-- VIDEO DE FONDO: autoplay + muted (sin controles) --> <video autoplay muted loop width="100%"> <source src="fondo.mp4" type="video/mp4"> </video>
AtributoEfectoCuándo usarlo
controlsMuestra play/pausa/volumen/barraSiempre que el usuario deba controlar
posterImagen que se ve antes de reproducirVideos informativos o personales
autoplay mutedInicia sin acción del usuario, sin sonidoSolo en videos decorativos de fondo
loopSe repite continuamenteFondos animados
Sin controls ni autoplay: El navegador no reproduce el video. El usuario lo ve estático y no puede hacer nada. Siempre pon controls, o autoplay+muted si es decorativo.
✏️

El <head> completo y CSS externo

El <head> es invisible para el usuario pero define cómo aparece la página en buscadores y dispositivos móviles. El CSS externo le da estilo coherente a todo el proyecto con un solo archivo.

meta charset · viewport · description · Google Fonts · link stylesheetAutoconciencia
9 / 10 · ¡Casi!

Al terminar esta guía puedes

  • Escribir un <head> completo con todas las metaetiquetas esenciales
  • Vincular un CSS externo con <link rel="stylesheet">
  • Escribir CSS básico que aplique a las 4 páginas del proyecto
🧠

El head completo + tu primer estilos.css

El <head> tiene tres funciones: decirle al navegador cómo renderizar (charset, viewport), decirle a los buscadores de qué trata (title, description), y cargar recursos externos (Google Fonts, CSS propio). El CSS en estilos.css se enlaza una vez y aplica a todas las páginas.

index.html → <head> completo
<head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Daniela Pérez · Estudiante ENSDB Barranquilla</title> <meta name="description" content="Mi página personal. Soy estudiante de décimo en la ENSDB."> <!-- Google Fonts --> <link rel="preconnect" href="https://fonts.googleapis.com"> <link href="https://fonts.googleapis.com/css2?family=Nunito:wght@400;700&display=swap" rel="stylesheet"> <!-- CSS propio —el que escribo en estilos.css --> <link rel="stylesheet" href="estilos.css"> </head>
estilos.css · nueva carpeta
body { font-family: 'Nunito', sans-serif; background: #f0f4f8; color: #1e293b; line-height: 1.7; max-width: 800px; margin: 0 auto; padding: 20px; } h1 { color: #1a4f8a; font-size: 2rem; } h2 { color: #2563eb; } nav a { margin-right: 16px; color: #2563eb; text-decoration: none; } nav a:hover { text-decoration: underline; } blockquote { border-left: 4px solid #2563eb; padding-left: 16px; font-style: italic; color: #475569; }
✏️
