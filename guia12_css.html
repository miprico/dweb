Colores, Fuentes, Texto y Box Model

Aprenderás a controlar el color del texto y el fondo, la tipografía, la alineación del texto y el Box Model completo: márgenes, rellenos, bordes, sombras y dimensiones. Y a través de eso, a reflexionar sobre cómo el espacio que le damos a las cosas — y a las personas — cambia completamente su impacto.

El espacio que le damos a las cosas — y a las personas

🖼️En casa · El espacio que hace la diferencia

La mamá de Valeria pegó las fotos de la familia directamente en la pared sin marcos, apretadas una contra otra. Se ven, pero nadie las mira. Un día reorganizaron todo: pusieron marcos, dejaron espacio entre cada foto, cambiaron el fondo a un color más cálido. Las mismas fotos. El espacio y el color cambiaron todo — las fotos ahora se pueden ver de verdad. En CSS, eso se llama Box Model: el espacio que rodea cada elemento lo define tanto como el elemento mismo.

💭 ¿Hay cosas en tu vida que no se "ven" bien porque no tienen el espacio que merecen?
🎨En el colegio · Los colores que comunican

En las evaluaciones de Sebastián, el docente siempre usa rojo para los errores y verde para los aciertos. Sebastián empieza a asociar el rojo con fracaso, aunque a veces el rojo solo marca una pregunta que faltó completar. Los colores no son neutros — comunican emociones antes de que el cerebro procese el texto. Cuando elegís un color en CSS, también estás eligiendo qué sensación producís en quien lo ve.

💭 ¿Hay un color que asocias con algo que viviste — positivo o negativo?
✍️Con compañeros · La letra que no se puede leer

En el grupo de trabajo, los documentos que prepara Lucía siempre son imposibles de leer: fuente muy pequeña, sin espacio entre líneas, texto pegado a los bordes. No es que Lucía no sepa escribir — es que nadie puede acceder a lo que escribe. La tipografía y el espaciado no son detalles decorativos: son accesibilidad. Un texto que no se puede leer es como una voz que no se escucha.

💭 ¿Hay alguien en tu entorno que tiene cosas importantes que decir pero no sabe cómo hacerse escuchar (o leer)?

Las fotos de Valeria necesitaban espacio. Sebastián aprendió que el rojo comunica algo antes del texto. Los documentos de Lucía necesitaban tipografía legible. En CSS, las propiedades de color, fuente, texto y Box Model son las herramientas que deciden si lo que creaste se puede ver, leer y sentir como merece.

01

Color y fondo: la emoción antes del texto

color · background-color — los dos primeros mensajes que recibe quien te lee

Explicación

color controla el color del texto. background-color controla el color de fondo del elemento. Los colores en CSS se pueden escribir de tres formas principales: nombre (red, blue, gold), hexadecimal (#1a4f8a) o RGB / RGBA (rgb(26, 79, 138) — la A de RGBA es opacidad).

Mismo color, tres formatos:

rgba(.4)
rgba(.15)
fondo
colores.css
/* color: el texto. background-color: el fondo */ body { color: #1e293b; /* texto oscuro */ background-color: #f0f4f8; /* fondo gris suave */ } h1 { color: #1a4f8a; /* azul institucional */ } .alerta { color: #991b1b; background-color: #fee2e2; /* rojo suave de fondo */ } .logro { color: #14532d; background-color: #dcfce7; /* verde suave de fondo */ } /* RGBA: mismo color con opacidad reducida */ .sombra-suave { background-color: rgba(26, 79, 138, 0.1); }

02

Fuentes y texto: la voz escrita

font-family · font-size · font-weight · line-height · text-align · text-decoration · text-transform

Explicación

La tipografía es la personalidad del texto. Una fuente redondeada y amigable transmite calidez. Una fuente técnica y monoespaciada transmite precisión. El tamaño, el peso y el espaciado entre líneas determinan si el texto invita a ser leído o lo ahuyenta.

font-family: 'Nunito', sans-serif
Lo que siento hoy
Redondeada, cálida. Ideal para contenido emocional o accesible.
font-family: 'Source Code Pro', monospace
Lo que siento hoy
Técnica, precisa. Ideal para código o contenido estructurado.
font-family: Georgia, serif
Lo que siento hoy
Clásica, literaria. Ideal para textos largos o citas.
font-family: Arial, sans-serif
Lo que siento hoy
Neutra, universal. La más común y la más segura.
tipografia.css
body { font-family: 'Nunito', sans-serif; /* fuente principal + fallback */ font-size: 15px; /* tamaño base */ line-height: 1.7; /* espacio entre líneas: 1.5–1.8 es ideal */ } h1 { font-size: 32px; font-weight: 900; /* 100(fino)→400(normal)→700(negrita)→900(extra) */ line-height: 1.2; /* encabezados con menos espacio entre líneas */ } p { text-align: left; /* left | center | right | justify */ } .cita { font-style: italic; text-align: center; } a { text-decoration: none; /* quita el subrayado por defecto */ } a:hover { text-decoration: underline; } .mayusculas { text-transform: uppercase; /* uppercase | lowercase | capitalize */ letter-spacing: 2px; /* espacio entre letras */ } /* text-shadow: offset-x offset-y blur color */ .con-sombra { text-shadow: 2px 2px 6px rgba(0,0,0,0.2); }

Regla del line-height: Para texto de lectura, usa entre 1.5 y 1.8. Para encabezados, entre 1.1 y 1.3. Un line-height: 1 es sofocante — las líneas se pegan. Un line-height: 2.5 se siente desconectado.
03

El Box Model: el espacio que define todo

margin · padding · border · width · height — cómo respira cada elemento en la página

Explicación

Todo elemento HTML es una caja con cuatro capas concéntricas. De adentro hacia afuera: el contenido (el texto o imagen), el padding (espacio entre el contenido y el borde), el border (la línea que delimita) y el margin (el espacio entre este elemento y los demás).

boxmodel.css
.tarjeta { /* DIMENSIONES */ width: 360px; /* ancho fijo */ max-width: 100%; /* nunca más ancho que su contenedor */ min-height: 120px; /* altura mínima aunque el contenido sea poco */ /* PADDING: espacio interior */ padding: 24px 28px; /* 24px arriba/abajo, 28px izq/der */ /* equivale a: padding-top, right, bottom, left */ /* BORDER: el borde visible */ border: 1.5px solid #e2e8f0; /* grosor tipo color */ border-radius: 14px; /* esquinas redondeadas */ /* MARGIN: espacio exterior */ margin: 0 auto; /* 0 arr/abajo, auto izq/der = centrado */ margin-bottom: 20px; /* solo abajo */ /* SOMBRA: profundidad */ box-shadow: 0 4px 14px rgba(0,0,0,.08); /* offset-x offset-y blur spread color */ background-color: #fff; }

Shorthand — escribir menos sin perder control

shorthand.css
/* PADDING shorthand — los 4 valores posibles */ .ejemplo { padding: 10px; } /* los 4 lados iguales */ .ejemplo { padding: 10px 20px; } /* vertical horizontal */ .ejemplo { padding: 10px 20px 15px; } /* top horizontal bottom */ .ejemplo { padding: 10px 20px 15px 5px; } /* top right bottom left (reloj) */ /* BORDER shorthand */ .btn { border: 2px solid #1a4f8a; } /* grosor tipo color */ .btn { border-top: 3px solid #fbbf24; } /* solo borde superior */ /* BOX-SHADOW puede tener múltiples sombras */ .card-premium { box-shadow: 0 2px 8px rgba(0,0,0,.06), 0 0 0 1px rgba(0,0,0,.04); }
box-sizing: border-box: Por defecto en CSS, el padding y el borde se suman al ancho. Si declaras width: 300px y luego padding: 20px, el elemento queda con 340px. Con box-sizing: border-box el padding queda dentro del ancho declarado. Agrega * { box-sizing: border-box; } al inicio de tu CSS siempre.

A1

La tarjeta que me representa: colores, fuente y espacio con intención

Individual · 25 minutos · Autoconciencia — que cada propiedad diga algo de ti
⏱ 25 min 👤 Individual 💻 HTML + CSS externo 🔍 SEL: Autoconciencia
🎨Antes de escribir — las preguntas de diseño

Antes de escribir una línea, responde en papel: ¿Qué color comunica cómo te sientes esta semana? ¿Cuánto espacio (padding) le das a lo que escribes — mucho aire o todo apretado? ¿El borde de tu tarjeta es sólido y firme, o suave y redondeado? Cada propiedad que elijas va a decir algo de ti — más de lo que crees.

💭 ¿Cuánto espacio te das a ti mismo — interior y exterior — para respirar?
  • 1

    Construye en HTML una tarjeta personal

    Con: <article class="mi-tarjeta"> que contenga un <h2> con algo que te define (no solo tu nombre), un <p class="descripcion"> con cómo te sientes esta semana, una <blockquote> con una frase que te sostiene, y un <a href="#">.

  • 2

    Estiliza la tarjeta con CSS usando las propiedades de hoy

    Mínimo obligatorio: color, background-color, font-family, font-size, font-weight, line-height, text-align, padding, margin, border, border-radius, box-shadow, width o max-width.

  • 3

    Aplica text-shadow al h2 con intención

    La sombra del texto del encabezado debe ser coherente con el tono de la tarjeta — suave si la tarjeta es tranquila, más pronunciada si es intensa.

  • 4

    Agrega comentarios CSS que expliquen las decisiones

    Antes de cada grupo de propiedades, un comentario que diga por qué elegiste ese color, ese padding, esa fuente. Las decisiones de diseño son decisiones personales y merecen justificación.

📋 Tu CSS debe aplicar
  • color y background-color con contraste suficiente para ser legible
  • font-family (Google Fonts), font-size, font-weight, line-height
  • text-align en al menos dos elementos distintos
  • padding en la tarjeta, margin para centrarla o separarla
  • border con border-radius redondeado
  • box-shadow con valores que den profundidad sin exagerar
  • text-shadow en el h2
  • Comentarios CSS que justifiquen cada decisión de diseño
🔍 SEL · Autoconciencia

Las propiedades de diseño son decisiones de identidad

Elegir el padding de una tarjeta no es un detalle técnico neutro. Es decidir cuánto espacio le das al contenido para respirar. Elegir el color es decidir qué emoción quieres provocar antes de que lean una sola palabra. La autoconciencia incluye "identificar las emociones propias" — y también las señales visuales que las comunican. En tu bitácora:

  • → ¿Qué color elegiste para tu tarjeta y qué dice de cómo te sientes esta semana?
  • → ¿Le diste mucho o poco padding? ¿Eso refleja cómo te das espacio a ti mismo en la vida real?
  • → ¿El borde de tu tarjeta es sólido o suave? ¿Eso dice algo de cómo estableces límites?
  • → ¿La tipografía elegida coincide con la "voz" que quieres que tenga tu contenido?
A2

El espacio que cambia el mensaje: margin y padding como comunicación

Parejas · 20 minutos · Conciencia social — cómo el espacio afecta lo que se recibe
⏱ 20 min 👥 Parejas 💻 Mismo HTML, márgenes diferentes 🌍 SEL: Conciencia Social
🖼️Volvamos a Valeria — las fotos sin y con espacio

Las fotos de la familia de Valeria eran las mismas — pero sin espacio entre ellas nadie las miraba. Con espacio, con marco, con aire, las mismas fotos se convirtieron en algo que la gente se detenía a ver. Esta actividad explora eso: tomar el mismo contenido HTML y cambiar solo el espacio — para demostrar que el margin y el padding no son detalles, son parte del mensaje.

💭 ¿Cuándo alguien o algo en tu vida necesita más espacio para ser visto de verdad?
  • 1

    Usen el mismo HTML base — sin cambiar nada

    Tres tarjetas con el mismo contenido: nombre, descripción de una emoción, una cita. Pueden ser los mismos o diferentes contenidos — lo importante es que el HTML sea idéntico en los dos archivos.

  • 2

    Cada uno crea su CSS con una filosofía de espacio opuesta (8 min)

    Persona A: mínimo espacio. Padding de 6px, margin de 4px, border sin radius, sin box-shadow. Todo apretado. Persona B: máximo espacio generoso. Padding de 32px, margin de 24px, border-radius de 20px, box-shadow suave. Todo con aire.

  • 3

    Comparen las dos versiones y conversen (7 min)

    ¿Cuál da más ganas de leer? ¿Cuál se siente más cuidada? ¿El contenido cambia según el espacio? ¿Cuál refleja mejor el respeto por lo que dice?

📋 Producto esperado A2
  • Dos archivos CSS opuestos para el mismo HTML
  • Un <p> reflexivo al final de cada archivo: qué comunica el espacio de tu versión
  • Una conclusión conjunta en un comentario CSS: ¿qué aprendieron sobre el espacio?
🌍 SEL · Conciencia Social

El espacio que le damos a las personas cambia lo que pueden mostrarnos

La conciencia social incluye "demostrar empatía y compasión". El espacio en CSS no es un lujo visual — es respeto por el contenido. En las relaciones, el espacio que le das a alguien para hablar, para equivocarse, para ser diferente, cambia lo que esa persona puede mostrarte. Un mensaje leído con espacio se entiende diferente que un mensaje leído apretado y sin aire. En tu bitácora:

  • → ¿Cuál de las dos versiones daba más ganas de leer el contenido? ¿Por qué?
  • → ¿Hay personas en tu vida a las que le das poco "espacio" para hablar o para ser? ¿Qué pasaría si les dieras más?
  • → ¿Hay contextos en tu vida donde sientes que el espacio es demasiado pequeño — donde no puedes ser tú mismo/a completamente?
A3

La guía de estilos del salón: color, tipografía y espaciado acordados

Grupos de 4 · 20 minutos · Toma de decisiones — crear un sistema de diseño colectivo
⏱ 20 min 👥 Grupos de 4 💻 Archivo de variables ⚖️ SEL: Toma de Decisiones
📐Un sistema de diseño que funcione para todos

Los equipos de diseño profesionales tienen una "guía de estilos" — una lista de colores, fuentes y espaciados que todos usan para que el producto sea consistente. Hoy el grupo va a construir la suya. El reto: ponerse de acuerdo en un sistema de diseño que todos sientan que los representa — no solo el gusto de quien habla más fuerte.

💭 ¿Qué colores y tipografía representarían al grupo completo — sin dejar a nadie por fuera?
  • 1

    Cada integrante propone en papel (3 min)

    Color principal, color secundario, color de fondo, fuente para encabezados, fuente para cuerpo de texto, padding base para tarjetas. Sin comparar todavía.

  • 2

    Construyan el archivo guia_estilos_salon.css con variables CSS (10 min)

    Usando el formato:

    guia_estilos_salon.css
    :root { --color-principal: #[elegido por consenso]; --color-secundario: #[elegido por consenso]; --color-fondo: #[elegido por consenso]; --color-texto: #[elegido por consenso]; --fuente-titulo: '[fuente elegida]', sans-serif; --fuente-cuerpo: '[fuente elegida]', sans-serif; --padding-base: [valor elegido]; --radio-borde: [valor elegido]; --sombra-base: 0 4px 14px rgba(0,0,0,.08); }
  • 3

    Documenten cada decisión (5 min)

    Antes de cada variable, un comentario CSS que diga por qué eligieron ese valor y si fue fácil o difícil ponerse de acuerdo en él.

📋 Producto esperado A3
  • Archivo guia_estilos_salon.css con mínimo 8 variables CSS
  • Comentarios CSS que expliquen el proceso de decisión de cada variable
  • Una regla CSS adicional al final que use todas las variables para demostrar que funcionan juntas
⚖️ SEL · Toma de Decisiones Responsables

Un sistema de diseño es un conjunto de decisiones responsables y sostenibles

La toma de decisiones responsables incluye "demostrar curiosidad y mentalidad abierta" y "evaluar consecuencias de las decisiones". Una guía de estilos no es caprichosa — es una herramienta que permite que el trabajo de varios sea coherente, que nadie pierda tiempo eligiendo colores de nuevo y que el producto final tenga calidad. En tu bitácora:

  • → ¿Cuál fue la variable más difícil de acordar? ¿Por qué generó más debate?
  • → ¿Hay alguna variable que quedó con un valor que no era tu primera opción? ¿Puedes vivir con esa decisión colectiva?
  • → ¿Cómo se parece este proceso a otras decisiones colectivas en tu vida (en la familia, en el colegio)?
  • → Si aplicaran esta guía de estilos al proyecto del salón de la Guía 09, ¿quedaría más o menos coherente que antes?

Criterios de valoración · Guía 12

¿Cómo se evaluará tu trabajo?

CriterioBásico (1–2)Satisfactorio (3)Superior (4–5)
Color y contraste
Técnico		 Los colores no tienen contraste suficiente o son aleatorios. Colores con contraste legible usando hex o rgba. La paleta es coherente, tiene contraste accesible y comunica algo intencional antes de que se lea el texto.
Tipografía y texto
Técnico		 Fuente por defecto, sin line-height ajustado, tamaño inadecuado. font-family, font-size, line-height y text-align aplicados correctamente. La tipografía es legible, bien espaciada y coherente con el tono del contenido.
Box Model completo
Técnico		 Faltan margin o padding, o el borde no está definido. margin, padding, border, border-radius y box-shadow aplicados. El Box Model está dominado: los espacios dan respiro al contenido y la tarjeta se siente bien construida.
Diseño con intención
SEL · Autoconciencia		 Las propiedades están puestas sin pensar — los comentarios dicen "puse esto aquí". Los comentarios CSS explican el por qué de las decisiones más importantes. Se nota que cada decisión (color, espacio, fuente) tiene una razón vinculada a la identidad o el tono del contenido.
Reflexión sobre espacio y personas
SEL		 La reflexión no conecta el Box Model con situaciones de vida real. Conecta el concepto de espacio CSS con algo personal. La reflexión muestra que el concepto de "espacio" — para el contenido y para las personas — es ahora una herramienta de pensamiento real.
