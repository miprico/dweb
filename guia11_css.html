¿Qué es CSS? Sintaxis, métodos y selectores

CSS es el lenguaje que le da forma, color y estilo al contenido HTML. Aprenderás cómo funciona, cómo se incluye y cómo seleccionar exactamente lo que quieres transformar. Y a través de eso, a reflexionar sobre algo que también aplica a las personas: la diferencia entre el contenido (lo que eres) y la forma en que te presentas al mundo (cómo te ven).

La forma en que te presentas — y el fondo que hay debajo

Lee estas tres situaciones antes de escribir una sola línea de CSS.

👗Con compañeros · La misma persona, diferente presentación

Mismo día, dos contextos diferentes. En el colegio, Camila llega de uniforme, callada, peinada. En el ensayo de danza de su barrio, Camila llega con ropa suya, habla fuerte, se ríe de todo. Es la misma persona — el mismo contenido. Pero la forma en que se presenta es completamente distinta según el contexto. CSS hace exactamente eso con el HTML: el contenido no cambia, pero la presentación sí.

💭 ¿Cómo cambias tú según el contexto — con tu familia, en el colegio, con tus amigos?
🏠En casa · Lo que se ve y lo que está debajo

La casa de Tomás por fuera se ve igual que todas las del barrio. Por dentro tiene las paredes pintadas del color favorito de cada quien, fotos que nadie más ha visto, un mural que él mismo hizo. La estructura es la misma (HTML). Lo que la hace única está en los detalles de presentación (CSS). Nadie lo sabe hasta que entra.

💭 ¿Hay algo de tu "interior" — de cómo eres por dentro — que la gente no ve porque la presentación externa no lo muestra?
🏫En el colegio · La especificidad de las reglas

En el colegio hay reglas generales: todos deben llegar a las 6:45. Pero hay reglas más específicas: el equipo de fútbol puede llegar a las 7:00 los martes de entrenamiento. Y hay reglas todavía más específicas: un estudiante con una condición médica documentada puede llegar más tarde. La regla más específica gana. En CSS se llama especificidad — y funciona exactamente igual.

💭 ¿Hay reglas en tu vida que se superponen? ¿Cuándo una regla más específica reemplaza a una general?

Camila es la misma en todos los contextos pero se presenta diferente. La casa de Tomás tiene una estructura estándar por fuera y algo único por dentro. Las reglas del colegio tienen niveles de especificidad. CSS funciona así: selecciona elementos, aplica reglas, y la regla más específica gana. En esta guía aprenderás cómo.

01

¿Qué es CSS? Sintaxis: selector, propiedad y valor

La gramática del estilo — tres palabras que cambian todo lo que se ve

Explicación

CSS (Cascading Style Sheets — Hojas de Estilo en Cascada) es el lenguaje que controla cómo se ve el contenido HTML en pantalla: colores, tipografías, tamaños, espacios, posiciones. Sin CSS, una página web es texto y estructura sin forma. Con CSS, esa misma estructura puede verse completamente diferente.

Toda regla CSS tiene exactamente tres partes: el selector (a qué elemento aplica), la propiedad (qué aspecto se cambia) y el valor (a qué se cambia).

02

Tres métodos para incluir CSS

En línea, interno y externo — igual que hay formas distintas de comunicar lo mismo

Explicación

Hay tres maneras de agregar CSS a una página HTML. Cada una tiene un contexto en el que tiene sentido — y todas afectan de manera diferente cuánto alcance tiene el estilo.

⚠️ Método 1 · En línea (inline)

style="" directamente en la etiqueta

El estilo va dentro del atributo style de un elemento específico. Afecta solo ese elemento. Difícil de mantener.

<p style="color:red">Texto</p>

Usar solo para casos muy específicos o pruebas rápidas.

⚡ Método 2 · Interno (<style>)

Dentro del <head> en una etiqueta <style>

El CSS va en el mismo archivo HTML. Afecta a toda esa página pero no a otras páginas del proyecto.

<style>
p { color: red; }
</style>

Útil para páginas únicas o prototipos rápidos.

✅ Método 3 · Externo (<link>)

Archivo .css separado enlazado con <link>

El CSS vive en su propio archivo. Se enlaza desde el <head>. Afecta a todas las páginas que lo enlacen. Es la mejor práctica.

<link rel="stylesheet"
href="estilos.css">

La forma correcta para proyectos reales. Ya la aprendiste en Guía 09.

La metáfora de los tres métodos: CSS en línea es como decirle algo a alguien susurrándole al oído — solo esa persona lo escucha. CSS interno es como hablar en un salón de clases — todos en ese espacio te escuchan. CSS externo es como publicar algo — cualquiera que lo encuentre puede aplicarlo. Cada forma tiene su contexto.

03

Selectores: apuntar exactamente a quien necesitas

No siempre quieres cambiar todo — a veces necesitas llegar a una sola persona

Explicación

Un selector define a qué elemento o elementos se aplica una regla CSS. Hay selectores que apuntan a todo un tipo de elemento, otros que apuntan a una clase específica, otros a un id único. Cuanto más específico el selector, más control tienes.

💡La especificidad explicada con el colegio

Si el rector dice "todos los estudiantes deben usar uniforme", eso aplica a todos — como el selector de elemento. Si el profesor de Educación Física dice "los estudiantes del grupo A usen sudadera hoy", eso aplica solo a ese grupo — como el selector de clase. Si la orientadora llama por nombre a un estudiante específico para una cita, solo ese va — como el selector de id. Cuanto más específico, más prioridad tiene.

selectores.css
/* Selector de elemento: aplica a TODOS los párrafos */ p { color: #334155; font-size: 15px; } /* Selector de clase: aplica a TODO elemento con class="importante" */ .importante { color: #dc2626; font-weight: 800; } /* Selector de id: aplica SOLO al elemento con id="titulo-principal" */ #titulo-principal { font-size: 28px; color: #1a4f8a; } /* Selector combinado: solo los p que están dentro de .tarjeta */ .tarjeta p { padding: 12px; background: #f8fafc; } /* Selector de hijo directo: solo li directos del ul */ ul > li { border-bottom: 1px solid #e2e8f0; } /* Selector de hermano adyacente: el primer p justo después de h2 */ h2 + p { font-size: 16px; color: #64748b; }

Pseudoclases y pseudoelementos

Las pseudoclases seleccionan un elemento en un estado especial — cuando el usuario pasa el cursor por encima (:hover), cuando está haciendo clic (:active), cuando un campo está enfocado (:focus). Los pseudoelementos seleccionan una parte virtual del elemento — el contenido antes (::before) o después (::after) del elemento real.

pseudo.css
/* :hover — cuando el cursor está encima del enlace */ a:hover { color: #1d4ed8; text-decoration: underline; } /* :focus — cuando un input está activo (el cursor está dentro) */ input:focus { border-color: #2563eb; outline: none; } /* :nth-child(2) — el segundo elemento de su padre */ li:nth-child(2) { background: #eff6ff; } /* ::before — inserta contenido antes del elemento */ .emocion::before { content: "→ "; color: #2563eb; font-weight: 800; } /* ::after — inserta contenido después del elemento */ .emocion::after { content: " ←"; color: #94a3b8; }

Especificidad: cuando dos reglas se contradicen

Cuando dos reglas apuntan al mismo elemento, el navegador usa la especificidad para decidir cuál gana. La regla con mayor especificidad se aplica. El orden es: ID > clase > elemento.

!important: Existe la declaración color: red !important que fuerza una regla a ganar sobre todo. Pero usarla en exceso es una mala práctica — es como gritar para que te escuchen en vez de construir confianza. Úsala solo cuando no hay otra solución.

Selectores de atributos

atributos.css
/* Selecciona todos los input de tipo texto */ input[type="text"] { border: 1.5px solid #e2e8f0; border-radius: 8px; padding: 8px 12px; } /* Selecciona enlaces que abren en nueva pestaña */ a[target="_blank"] { color: #7c3aed; } /* Selecciona cualquier elemento con el atributo disabled */ [disabled] { opacity: 0.5; cursor: not-allowed; }
A1

Mi hoja de estilos: darle forma a quien soy

Individual · 25 minutos · Autoconciencia — elegir colores, fuentes y espacios que me representen
⏱ 25 min 👤 Individual 💻 Editor de código 🔍 SEL: Autoconciencia
🎨Antes de escribir — define quién eres visualmente

Antes de escribir una sola regla CSS, responde en papel: ¿Qué color te representa? No el que más te gusta en ropa — el que sientes que dice algo de tu personalidad. ¿Qué fuente se parece a cómo piensas — elegante y seria, o amigable y redondeada, o técnica y precisa? ¿Cuánto espacio necesitas — apretado o con mucho aire? Esas respuestas son tu sistema de diseño personal.

💭 ¿Si tu personalidad fuera un color y una tipografía, cuáles serían?
  • 1

    Retoma tu proyecto de las Guías 01–10

    Abre el estilos.css de tu proyecto personal. Esta actividad va a profundizar y mejorar ese archivo con todo lo que aprendiste hoy.

  • 2

    Define variables CSS para tu paleta personal

    Al inicio del archivo, declara al menos 5 variables con :root { --mi-variable: valor; }: tu color principal, color secundario, color de texto, color de fondo y tu fuente. Estas variables deben decir algo de ti — no son genéricas.

  • 3

    Usa los tres tipos de selector

    Escribe al menos: 3 reglas con selector de elemento, 3 con selector de clase, 1 con selector de ID. Usa también al menos un selector combinado (header p, nav a), una pseudoclase (:hover) y un pseudoelemento (::before o ::after).

  • 4

    Aplica los tres métodos — aunque sea como prueba

    En una de tus páginas HTML, añade temporalmente un estilo en línea a un elemento para ver la diferencia. Luego muévelo al CSS externo y observa qué cambia. Finalmente añade un bloque <style> interno con una regla de prueba. Observa qué prevalece.

📋 Tu estilos.css debe tener
  • Variables CSS en :root para colores y fuente
  • Al menos 3 selectores de elemento, 3 de clase, 1 de ID
  • Al menos 1 selector combinado y 1 pseudoclase (:hover mínimo)
  • Al menos 1 pseudoelemento (::before o ::after)
  • Comentarios CSS que expliquen cada grupo de reglas
  • Consistencia visual entre las 4 páginas del proyecto
🔍 SEL · Autoconciencia

Elegir los colores y la tipografía de tu página es elegir cómo presentarte al mundo

La autoconciencia incluye "identificar los bienes personales, culturales y lingüísticos". Las decisiones de diseño visual son también decisiones de identidad — qué colores usas, cuánto espacio dejas, qué fuente eliges. Dicen algo de ti. En tu bitácora:

  • → ¿Por qué elegiste esos colores? ¿Qué dicen de ti?
  • → ¿La presentación visual de tu página se parece a cómo te presentas en la vida real o es diferente? ¿Por qué?
  • → ¿Hay algo de tu personalidad que quisiste mostrar visualmente y que crees que lograste? ¿Y algo que no?

El mismo HTML, estilos completamente distintos

Parejas · 20 minutos · Conciencia social — el mismo contenido, leído de formas diferentes
⏱ 20 min 👥 Parejas 💻 Mismo HTML, dos CSS 🌍 SEL: Conciencia Social
🪞Volvamos a Camila — la misma persona, presentada diferente

Camila es la misma en el colegio y en el ensayo de danza. Pero quienes solo la conocen de un contexto tienen una imagen incompleta. Esta actividad propone usar el mismo HTML y crear dos hojas de estilo completamente diferentes — para ver cómo el CSS cambia radicalmente la percepción sin cambiar nada del contenido.

💭 ¿Cómo cambia la percepción que tienen de ti las personas que solo te conocen en un contexto?
  • 1

    Usen este HTML de base — el mismo para los dos

    base_compartida.html
    <article class="tarjeta"> <h2 id="nombre">Quién soy</h2> <p class="descripcion">Estudiante de décimo. Me gusta crear cosas.</p> <p class="emocion">Lo que siento hoy: motivación mezclada con cansancio.</p> <a href="#">Conocerme más</a> </article>
  • 2

    Cada uno crea su propio CSS para ese HTML (10 min)

    Sin ver el del otro. Usando el mismo HTML base. Pueden usar colores, tipografías, backgrounds, márgenes, bordes — todo lo que quieran. El objetivo es que los dos resultados visuales sean completamente distintos.

  • 3

    Comparen los resultados en el navegador (5 min)

    Abran los dos archivos. ¿Son reconocibles como el mismo HTML? ¿Qué emociones transmite cada versión? ¿Cuál parece más "tú"?

📋 Cada CSS debe tener
  • Estilos para .tarjeta, #nombre, .descripcion, .emocion y a
  • Al menos un :hover en el enlace
  • Al menos un .emocion::before con content
  • Variables CSS para los colores principales
🌍 SEL · Conciencia Social

El mismo texto se percibe completamente diferente según cómo se presenta

La conciencia social incluye "tomar la perspectiva de otros". La misma oración — "lo que siento hoy: motivación mezclada con cansancio" — con un CSS de colores cálidos y tipografía redondeada se siente cercana y humana. Con un CSS frío y tipografía monoespaciada se siente distante. El medio cambia el mensaje. En tu bitácora:

  • → ¿Cómo transmitían emociones distintas las dos versiones del mismo HTML?
  • → ¿Hay personas en tu vida que tienen una "presentación" que no refleja bien su contenido — personas que parecen difíciles de acercarse pero en realidad son cálidas, o viceversa?
  • → ¿Cuándo tu propia "presentación" exterior no refleja cómo eres por dentro?
A3

El CSS del salón: especificidad y consenso

Grupos de 4 · 20 minutos · Habilidades relacionales — qué regla prevalece cuando hay conflicto
⏱ 20 min 👥 Grupos de 4 💻 Archivo colectivo 💬 SEL: Habilidades Relacionales
⚖️La especificidad aplicada al grupo

En un grupo siempre hay "reglas" implícitas — cómo se resuelven los conflictos, quién tiene más peso en las decisiones, qué norma prevalece cuando dos personas quieren cosas distintas. En CSS, la especificidad resuelve esos conflictos de forma predecible. Esta actividad propone que el grupo use CSS para explorar ese tema: ¿qué pasa cuando dos reglas se contradicen?

💭 En tu grupo, cuando hay desacuerdo, ¿qué regla implícita resuelve el conflicto?
  • 1

    Cada integrante propone 3 reglas CSS para el salón (3 min)

    Para el mismo HTML del ejercicio anterior. Cada uno las escribe en papel. Las reglas deben reflejar su personalidad en el diseño.

  • 2

    Reúnan las 12 reglas en un solo CSS (8 min)

    Inevitablemente habrá conflictos: dos personas definieron el color del h2 de manera diferente, o el tamaño del texto. Deben resolver cada conflicto de una de estas formas: a) discutir y elegir una, b) usar especificidad (el selector más específico gana), c) usar !important — pero justificando por qué.

  • 3

    Documenten cada conflicto y su resolución

    En comentarios CSS, antes de cada regla que fue disputada, escriban: quién la propuso, cuál era la alternativa y por qué prevalece esta.

  • 4

    Reflexión grupal (5 min)

    ¿El proceso de resolver los conflictos CSS fue parecido a cómo resuelven los conflictos reales en el grupo? ¿Qué "especificidad" tienen las distintas voces en su grupo — quién tiene más peso y por qué?

📋 Producto esperado A3
  • Archivo CSS colectivo con las reglas de los 4 integrantes integradas
  • Comentarios CSS que documenten cada conflicto y cómo se resolvió
  • Al menos un uso consciente de especificidad para resolver un conflicto
  • Una reflexión grupal en el footer del HTML sobre el proceso
💬 SEL · Habilidades Relacionales

Resolver conflictos de especificidad en CSS es practicar resolución de conflictos reales

Las habilidades relacionales incluyen "resolver los conflictos constructivamente". Cuando dos reglas CSS apuntan al mismo elemento, el sistema tiene reglas claras para resolverlo — y esas reglas son predecibles, no arbitrarias. En las relaciones humanas, los conflictos se resuelven mejor cuando también hay reglas claras y conocidas. En tu bitácora:

  • → ¿Qué regla del grupo prevaleció más seguido? ¿Por qué crees que fue esa?
  • → ¿Alguien usó !important para forzar su regla? ¿Cómo se sintió el grupo con eso?
  • → ¿Hay conflictos en tu vida real donde sería útil tener una "especificidad" clara para resolverlos?
  • → ¿El proceso de hoy fue más difícil técnicamente o relacionalmente?

Criterios de valoración · Guía 11

¿Cómo se evaluará tu trabajo?

CriterioBásico (1–2)Satisfactorio (3)Superior (4–5)
Sintaxis CSS correcta
Técnico		 Las reglas CSS tienen errores de sintaxis (faltan llaves, punto y coma). Todas las reglas tienen sintaxis correcta: selector, propiedad y valor bien escritos. Sintaxis perfecta con variables CSS aplicadas consistentemente.
Variedad de selectores
Técnico		 Solo usa selectores de elemento o solo de clase. Usa los tres tipos (elemento, clase, ID) más un combinado y una pseudoclase. Usa todos los tipos incluyendo pseudoelementos y selectores de atributo. Se nota que eligió el selector correcto para cada caso.
Identidad visual
SEL · Autoconciencia		 Los colores y tipografía son genéricos o aleatorios. La paleta y tipografía son coherentes y muestran una elección intencional. El CSS transmite algo de la personalidad del estudiante. Se puede decir "esto se parece a esa persona".
Reflexión sobre forma y fondo
SEL		 La reflexión no conecta CSS con situaciones de vida real. Conecta la diferencia entre HTML (contenido) y CSS (presentación) con algo personal. La reflexión muestra comprensión real de cuándo la "presentación" ayuda y cuándo limita — en código y en la vida.
Trabajo colectivo
SEL · Habilidades Relacionales		 No participó o impuso sus reglas sin proceso de consenso. Participó y los conflictos de especificidad están documentados en comentarios. El proceso de resolución de conflictos CSS refleja madurez relacional — hay negociación genuina, no imposición.
