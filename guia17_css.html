El movimiento, la transformación y la memoria del color

Lee estas tres situaciones. Son las últimas de este proyecto.

🌊Con compañeros · El cambio que se nota — y el que no

Cuando Julia llegó a décimo, era muy callada. Al final del año, habla en clase, lidera proyectos, se ríe fuerte. Sus compañeros dicen que "cambió". Ella dice que no cambió — que siempre fue así, solo que ahora tiene el espacio para mostrarlo. La transición fue suave, gradual. No hubo un punto de quiebre — fue un movimiento constante hacia algo que ya estaba ahí. En CSS, transition hace exactamente eso: el cambio ocurre, pero de forma fluida, no abrupta.

💭 ¿Hay algo en ti que "siempre estuvo ahí" pero que tardó en tener el espacio para mostrarse?
📸En el colegio · La foto en blanco y negro

En el periódico mural de la ENSDB, publicaron las fotos del anuario en blanco y negro. Alguien dijo que sin color se veían más "serias". Pero Nicolás pensó lo contrario: sin color, las fotos perdían algo esencial — el contexto, la energía, lo que hace que cada quien sea quien es. filter: grayscale(100%) hace exactamente eso: elimina la información del color. Y a veces, cuando le quitamos el "color" a las personas — sus particularidades, su contexto — también las perdemos.

💭 ¿Qué sería perder "el color" para ti — qué le da vida a quien eres que no podría quitarse sin perder algo esencial?
🎨En casa · Un color que atraviesa todo

La mamá de Renata tiene un azul favorito. Está en las paredes del cuarto, en las cobijas, en la taza que usa todas las mañanas. No es una decisión consciente cada vez — ese azul simplemente aparece porque es su color. Si ella decorara su casa con variables CSS, --color-principal: #1a6fa8 aparecería en cada rincón. Las variables CSS son exactamente eso: un valor que defines una vez y que se propaga por todo el código, coherente y constante.

💭 ¿Hay algún "color" en tu vida — un valor, una forma de ser, una costumbre — que aparece en todos los contextos porque simplemente eres así?

Julia que cambia suavemente sin romperse. Las fotos que pierden algo cuando se les quita el color. El azul de la mamá de Renata que aparece en todos lados. Esta guía trata sobre lo que le da vida al código — y también sobre lo que le da vida a las personas.

01

transition y transform: movimiento con control

El cambio puede ser brusco — o puede ser suave. Esa elección también es una decisión.

Explicación

transition hace que los cambios de una propiedad CSS ocurran gradualmente en lugar de instantáneamente. Sin transition, al hacer hover en un botón el color cambia de golpe. Con transition, cambia suavemente en el tiempo que definas. transform mueve, rota, escala o sesga un elemento sin afectar el flujo de la página.

Prueba pasando el cursor por estos elementos:

Tarjeta con
scale + shadow

transition-transform.css
/* transition: propiedad duración timing-function delay */ .btn { background: #1a4f8a; color: white; padding: 10px 24px; border-radius: 8px; transition: all 0.3s ease; /* propiedad: "all" = todo | duración: 0.3s | timing: ease */ } .btn:hover { background: #2563eb; transform: translateY(-3px); /* sube 3px */ box-shadow: 0 8px 20px rgba(37,99,235,.3); } /* TRANSFORM: mover, rotar, escalar */ .escalar { transform: scale(1.1); } /* 10% más grande */ .rotar { transform: rotate(45deg); } /* gira 45 grados */ .mover { transform: translateX(20px); } /* mueve 20px a la derecha */ .mover-xy { transform: translate(10px, -5px); } /* Combinados: primero se aplica el último declarado */ .combo:hover { transform: scale(1.05) rotate(2deg); transition: transform 0.25s ease; } /* timing-function: cómo se acelera/desacelera */ /* ease (por defecto) | linear | ease-in | ease-out | ease-in-out */ /* cubic-bezier(x1,y1,x2,y2) para curvas personalizadas */

✨ Combinado
transform no rompe el flujo: Cuando usas transform: translateX(20px), el elemento se mueve visualmente pero su "huella" en el flujo sigue en el lugar original — los elementos vecinos no se mueven. Es diferente a cambiar margin-left, que sí afecta el flujo. Usa transform para animaciones de hover para no "desorganizar" el layout.
02

@keyframes: definir el movimiento fotograma a fotograma

Una animación es una secuencia de momentos — como la transformación de Julia a lo largo del año

Explicación

@keyframes define los momentos clave de una animación: qué aspecto tiene el elemento al 0%, al 50%, al 100% del recorrido. Luego se aplica al elemento con la propiedad animation. A diferencia de transition (que reacciona a un evento como hover), las animaciones pueden ejecutarse automáticamente, repetirse y tener múltiples pasos.

keyframes.css
/* 1. Definir la animación con @keyframes */ @keyframes pulso { 0%, 100% { transform: scale(1); opacity: 1; } 50% { transform: scale(1.08); opacity: .85; } } @keyframes deslizar { from { transform: translateX(-20px); opacity: 0; } to { transform: translateX(0); opacity: 1; } } @keyframes cambio-color { 0% { background: #1a4f8a; } 33% { background: #7c3aed; } 66% { background: #ea580c; } 100% { background: #1a4f8a; } } /* 2. Aplicar la animación al elemento */ .pulsante { animation-name: pulso; animation-duration: 2s; animation-timing-function: ease-in-out; animation-iteration-count: infinite; /* infinite o un número */ animation-direction: alternate; /* va y vuelve */ animation-delay: 0.5s; /* empieza después de 0.5s */ /* Shorthand: nombre duración timing-function delay iteraciones */ animation: pulso 2s ease-in-out infinite; } .entrada { animation: deslizar 0.5s ease forwards; /* forwards: al terminar, queda en el estado final */ } /* animation-fill-mode: */ /* forwards: queda en el 100% al terminar */ /* backwards: aplica el 0% antes de empezar (con delay) */ /* both: forwards + backwards */
PropiedadQué controlaValores comunes
animation-nameQué @keyframes usaEl nombre que diste al @keyframes
animation-durationCuánto dura un ciclo0.3s · 1s · 2.5s
animation-timing-functionAceleración del movimientoease · linear · ease-in-out
animation-iteration-countCuántas veces se repite1 · 3 · infinite
animation-delayCuánto espera antes de empezar0s · 0.5s · 1s
animation-fill-modeQué pasa antes/despuésforwards · backwards · both

Filtros, imágenes de fondo y variables CSS

El toque final: efectos visuales y el sistema de diseño coherente

Explicación

filter — efectos visuales sobre elementos

filter aplica efectos visuales a un elemento: desenfoque, escala de grises, brillo, contraste, saturación. Se puede animar con transition. Pasa el cursor por cada caja:

filtros.css
/* filter: efectos visuales */ .foto { filter: grayscale(100%); } /* blanco y negro */ .foto:hover { filter: grayscale(0%); } /* color al hover */ .foto { transition: filter 0.4s ease; } .desenfocado { filter: blur(4px); } .brillante { filter: brightness(1.3); } .oscuro { filter: brightness(0.7); } .contraste { filter: contrast(1.5); } .sepia { filter: sepia(80%); } /* Combinados */ .foto-vintage { filter: sepia(60%) contrast(1.1) brightness(0.9); } /* background-image: imagen de fondo */ .hero { background-image: url('mi-imagen.jpg'); background-size: cover; /* cubre todo sin distorsionar */ background-position: center; /* centrada */ background-repeat: no-repeat; min-height: 400px; } /* Gradiente como background-image */ .gradiente { background-image: linear-gradient(135deg, #1a4f8a, #7c3aed); } /* Imagen con overlay oscuro para leer texto encima */ .foto-con-texto { background-image: linear-gradient(rgba(0,0,0,.5), rgba(0,0,0,.5)), url('mi-foto.jpg'); background-size: cover; }

Variables CSS — el color que aparece en todos lados

Las variables CSS (también llamadas custom properties) permiten guardar un valor con un nombre y reutilizarlo en todo el CSS. Se declaran con --nombre-variable: valor y se usan con var(--nombre-variable). Definidas en :root, están disponibles en toda la página.

variables.css
/* Declarar variables en :root = disponibles en TODA la página */ :root { /* Colores */ --color-principal: #1a4f8a; --color-secundario: #2563eb; --color-acento: #fbbf24; --color-fondo: #f0f4f8; --color-texto: #1e293b; /* Tipografía */ --fuente-principal: 'Nunito', sans-serif; --tamaño-base: 1rem; /* Espaciado */ --espacio-sm: 0.5rem; --espacio-md: 1rem; --espacio-lg: 2rem; --radio-borde: 14px; --sombra-base: 0 2px 12px rgba(0,0,0,.06); /* Transiciones */ --transicion: all 0.3s ease; } /* Usar con var() */ body { font-family: var(--fuente-principal); color: var(--color-texto); background: var(--color-fondo); } .card { background: white; border-radius: var(--radio-borde); padding: var(--espacio-lg); box-shadow: var(--sombra-base); transition: var(--transicion); } .card:hover { box-shadow: 0 8px 24px rgba(0,0,0,.12); transform: translateY(-4px); } /* Cambiar el tema completo cambiando una sola variable */ @media (prefers-color-scheme: dark) { :root { --color-fondo: #0f172a; --color-texto: #e2e8f0; } }

El poder de las variables CSS: Si defines --color-principal: #1a4f8a y lo usas en 50 lugares, para cambiar el color de todo el proyecto solo cambias esa variable. Sin variables, cambiarías los 50 valores manualmente. Es igual que tener un valor que defines una vez — como el azul de la mamá de Renata — y que aparece coherentemente en todos lados.

A1

Darle vida a mi proyecto: animaciones y variables definitivas

Individual · 25 minutos · Autoconciencia — el toque final que hace que algo sea completamente tuyo
⏱ 25 min 👤 Individual 💻 CSS externo 🔍 SEL: Autoconciencia
✨El toque final — lo que hace que algo sea tuyo

La transición suave de Julia, el color que la mamá de Renata pone en todos lados, la diferencia entre una foto en color y en blanco y negro. Esta actividad es el toque final de todo el proyecto: las animaciones, los filtros y las variables que lo hacen inconfundiblemente tuyo.

💭 ¿Qué animación o efecto visual diría algo de tu personalidad — de cómo te mueves por el mundo?
  • 1

    Consolida todas las variables CSS en el :root de tu estilos.css

    Revisa todo tu CSS y extrae los colores, fuentes, espaciados y radios repetidos. Conviértelos en variables con nombres descriptivos. Luego reemplaza todos los valores hardcoded por var(--nombre). Al terminar, el tema visual de toda la página debe controlarse desde el bloque :root.

  • 2

    Agrega transition a todos los elementos interactivos

    Botones, links del nav, tarjetas, imágenes — todo lo que tenga :hover debe tener transition. Usa el valor guardado en variable: transition: var(--transicion).

  • 3

    Crea una animación @keyframes que represente algo de ti

    No tiene que ser compleja. Puede ser un pulso suave en tu h1 al cargar la página, una entrada deslizada para las tarjetas, una rotación lenta en un ícono. El nombre del @keyframes debe decir algo de lo que hace (entrada-suave, latido, giro-lento).

  • 4

    Aplica un efecto filter a las fotos o imágenes

    Las imágenes en estado normal pueden tener filter: grayscale(20%) (ligeramente desaturadas). Al hover, filter: grayscale(0) (color completo). Añade transition: filter 0.4s ease. Es el efecto "la foto cobra vida al mirarla".

📋 Tu CSS final debe tener
  • Bloque :root completo con todas las variables del proyecto
  • transition en todos los elementos interactivos usando la variable --transicion
  • Al menos un @keyframes personalizado con un nombre que diga algo
  • filter aplicado a imágenes con transition
  • background-image en al menos una sección (imagen real o gradiente)
  • Comentarios CSS que expliquen qué representa cada animación
🔍 SEL · Autoconciencia

Las animaciones son la personalidad del código — lo que lo hace único

Un sitio web sin animaciones funciona. Un sitio con animaciones bien elegidas tiene carácter. Como las personas: funcionar no es suficiente — lo que nos hace reconocibles es la forma en que nos movemos, la consistencia de nuestros "colores", la suavidad o la energía con que cambiamos. En tu bitácora:

  • → ¿Qué @keyframes creaste y qué dice de ti? ¿Por qué esa animación y no otra?
  • → ¿Las variables CSS de tu proyecto tienen nombres que reflejan algo tuyo, o son genéricos?
  • → ¿La transition que elegiste es rápida o lenta? ¿Eso dice algo de cómo cambias tú?
  • → ¿Hay algo de tu código que ya, antes de que nadie lo lea, dice "esto lo hizo esa persona"?

Grayscale y color: lo que se pierde y lo que permanece

Parejas · 20 minutos · Conciencia social — ver qué se pierde cuando quitamos el "color" a las personas
⏱ 20 min 👥 Parejas 💻 + conversación 🌍 SEL: Conciencia Social
📸Volvamos a las fotos de Nicolás — lo que se pierde en blanco y negro

Nicolás sentía que las fotos en blanco y negro perdían algo esencial. Esta actividad propone convertir en código esa reflexión: construir una página donde las tarjetas de los integrantes del grupo aparecen en escala de grises y cobran color al ser "vistas" (hover). La metáfora es exacta: algunas personas solo "cobran color" cuando alguien se detiene a mirarlas de verdad.

💭 ¿Hay alguien en tu entorno que necesita ser "mirado de verdad" para mostrar su color real?
  • 1

    Retomen la página del equipo de guías anteriores

    O creen una nueva con tarjetas de los dos integrantes. Cada tarjeta tiene imagen de fondo (o un fondo de color), nombre y una frase.

  • 2

    Implementen el efecto grayscale → color (6 min)

    Estado base: filter: grayscale(100%) brightness(0.85). Estado hover: filter: grayscale(0%) brightness(1). Transition de 0.5s ease. El resultado: al pasar el cursor, la tarjeta "cobra vida".

  • 3

    Cada uno escribe sobre la metáfora (8 min)

    En el HTML, bajo cada tarjeta, un <p> que responde: "Lo que se pierde de [nombre] cuando nadie se detiene a verlo/a de verdad es..." Honesto, concreto, real.

📋 Producto esperado A2
  • Tarjetas con el efecto grayscale → color al hover con transition suave
  • Un párrafo debajo de cada tarjeta con la reflexión sobre "lo que se pierde"
  • Una conclusión conjunta: ¿qué aprendieron de verse a sí mismos en "blanco y negro"?
🌍 SEL · Conciencia Social

Algunas personas solo muestran su color cuando alguien se detiene a mirarlas

La conciencia social incluye "demostrar empatía y compasión" y "reconocer las fortalezas de los demás". El efecto grayscale → color no es solo un truco de CSS — es una metáfora real: hay personas en nuestro entorno que aparecen "en gris" para la mayoría pero que tienen un color completo cuando alguien se detiene. En tu bitácora:

  • → ¿Qué pusiste en "lo que se pierde de mí cuando nadie se detiene a verme"?
  • → ¿Hay alguien en tu entorno — familia, colegio, grupo — que vive en "grayscale" para la mayoría y que tú puedes "mirar" de verdad?
  • → ¿Hubo algo del párrafo del otro que te sorprendió — algo que no sabías que se perdía de ellos?

Criterios de valoración · Guía 17 · Cierre

¿Cómo se evaluará tu trabajo final?

CriterioBásico (1–2)Satisfactorio (3)Superior (4–5)
Transitions y transforms
Técnico		 Transition sin timing o aplicada solo a un elemento. Transition en todos los elementos interactivos con timing apropiado. La variable --transicion se usa consistentemente. Los transforms elegidos tienen intención: qué sube, qué escala, qué rota tiene sentido de diseño.
@keyframes personalizado
Técnico		 No usa @keyframes o copia uno genérico sin modificar. Un @keyframes propio bien escrito con al menos dos estados. El nombre del @keyframes y su comportamiento dicen algo de la personalidad del proyecto. animation-fill-mode y delay usados con criterio.
Variables CSS completas
Técnico		 Algunas variables pero valores hardcoded mezclados sin criterio. El :root tiene todas las variables del sistema y se usan consistentemente. Las variables tienen nombres descriptivos y coherentes. Cambiar una sola variable del :root cambia el tema completo.
Identidad visual del proyecto
SEL · Autoconciencia		 El proyecto tiene buen código pero podría ser de cualquier persona. El proyecto tiene una paleta y un carácter visual reconocible. El proyecto es inconfundiblemente de esa persona: las animaciones, los colores, las transiciones y el layout cuentan una historia coherente de quién lo hizo.
