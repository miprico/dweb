Aprenderás cómo CSS controla dónde están las cosas, cómo se comportan en el flujo de la página y qué pasa cuando el contenido no cabe. Y a través de eso, a reflexionar sobre los lugares que ocupamos — o no nos dejan ocupar — en la familia, el colegio y el grupo, y qué pasa cuando lo que sentimos no cabe en el espacio que nos dan.

01

position: dónde está cada elemento en la página

static · relative · absolute · fixed · sticky — cinco maneras de ocupar un lugar

Explicación

Por defecto, todos los elementos HTML siguen el flujo normal del documento — se apilan uno debajo del otro. La propiedad position rompe ese flujo y permite ubicar un elemento exactamente donde lo necesitas, usando las propiedades top, right, bottom y left.

posicionamiento.css
/* RELATIVE: se desplaza de su posición original */ .mi-lugar { position: relative; top: 10px; /* se baja 10px de donde estaría normalmente */ left: 20px; /* se mueve 20px a la derecha */ } /* ABSOLUTE: relativo al padre posicionado */ .contenedor { position: relative; /* el padre necesita position para que absolute funcione */ } .badge { position: absolute; top: -8px; right: -8px; /* esquina superior derecha del padre */ } /* FIXED: siempre visible, no se mueve al hacer scroll */ .nav-fijo { position: fixed; top: 0; left: 0; width: 100%; z-index: 100; /* siempre encima de todo */ } /* STICKY: se pega al hacer scroll hasta un umbral */ .encabezado-seccion { position: sticky; top: 60px; /* se pega cuando llega a 60px del tope */ background: #fff; z-index: 10; }

z-index: quién está encima de quién

z-index controla el orden de apilamiento de los elementos posicionados. Un z-index mayor significa que el elemento aparece encima de los demás. Solo funciona con elementos que tienen un position distinto de static.

z-index.css
.modal { position: fixed; z-index: 1000; } /* encima de todo */ .nav-fijo { position: fixed; z-index: 100; } /* siempre visible */ .tarjeta { position: relative; z-index: 10; } /* sobre el fondo */ .fondo { position: absolute; z-index: 1; } /* debajo de todo */

La metáfora del z-index: En la vida también existe el z-index — quién tiene más "capas" sobre quién. El director está encima del docente en la jerarquía. El docente encima del estudiante. Pero en una conversación genuina, esas capas pueden invertirse: el que sabe más sobre su propia experiencia siempre tiene z-index superior en su propio territorio.

02

display: cómo ocupa espacio un elemento

block · inline · inline-block · none — cuatro formas de estar o no estar en el flujo

Explicación

La propiedad display determina cómo un elemento participa en el flujo de la página. ¿Ocupa toda la línea? ¿Solo su contenido? ¿O desaparece completamente? Esta propiedad es fundamental para el diseño de cualquier interfaz.

Así se comportan los diferentes valores de display:

display: block — ocupa toda la línea disponible:

display: inline — solo ocupa lo que necesita, sin romper el flujo:

display: inline-block — en línea pero con ancho y alto propios:

display.css
/* block: ocupa todo el ancho, empieza en nueva línea */ /* Elementos block por defecto: div, p, h1-h6, ul, li, section */ div { display: block; } /* inline: solo ocupa lo que necesita, fluye con el texto */ /* Elementos inline por defecto: span, a, strong, em, img */ span { display: inline; } /* inline-block: en línea pero acepta width y height */ /* Útil para botones o items de menú horizontales */ .btn { display: inline-block; width: 140px; padding: 10px 20px; text-align: center; } /* none: desaparece COMPLETAMENTE — no ocupa espacio */ /* Como si el elemento no existiera en el HTML */ .oculto { display: none; } /* Ejemplo de uso real: menú móvil que se abre con JS */ .menu-movil { display: none; } .menu-movil.abierto { display: block; }

display: none vs visibility: hidden: Ambas "ocultan" un elemento, pero de manera diferente. display: none elimina el elemento completamente del flujo — como Alejandro cuando no participa: no ocupa espacio, nadie sabe que podría estar ahí. visibility: hidden lo hace invisible pero sigue ocupando espacio — como cuando estás presente en cuerpo pero mentalmente en otro lado.

03

overflow: qué pasa cuando el contenido no cabe

visible · hidden · scroll · auto — cuatro formas de manejar lo que desborda

Explicación

Cuando el contenido de un elemento es más grande que su contenedor, se "desborda" (overflow). La propiedad overflow controla qué sucede con ese contenido que sobra. Como Marisol cuando lo que siente no cabe en el espacio que tiene disponible — hay distintas formas de gestionarlo.

overflow: visible (por defecto)

El contenido desborda sin restricciones. Todo se ve aunque salga de la caja. Como cuando las emociones salen sin filtro aunque el contexto no sea el adecuado.

overflow: hidden

El contenido que no cabe se corta. No hay scroll. Lo que desborda desaparece. Como guardarse todo por dentro — lo que no cabe, no se ve.

overflow: scroll

Aparece barra de desplazamiento siempre, aunque no haga falta. El usuario puede hacer scroll para ver el contenido que no cabe. Siempre existe la barra aunque el contenido sea corto.

overflow: auto (recomendado)

Solo aparece barra de scroll cuando hace falta. Si el contenido cabe, no hay barra. Es la opción más inteligente y la más común en diseño real. El scroll aparece solo cuando el contenido lo necesita.
overflow.css
/* overflow controla el contenido que desborda la caja */ .caja-texto { width: 300px; height: 120px; overflow: auto; /* scroll solo si hace falta — lo más común */ } /* overflow-x y overflow-y para controlar cada eje */ .galeria { overflow-x: auto; /* scroll horizontal para ver más imágenes */ overflow-y: hidden; /* sin scroll vertical */ white-space: nowrap; /* los items no se van a la línea siguiente */ } /* text-overflow: qué pasa con el texto en una sola línea */ .texto-corto { overflow: hidden; white-space: nowrap; text-overflow: ellipsis; /* añade "..." al final del texto cortado */ } /* visibility: elemento invisible pero ocupa espacio */ .presente-pero-invisible { visibility: hidden; /* la caja sigue ahí, el contenido no se ve */ }

La metáfora de Marisol: overflow: visible sería dejar que todo salga sin control. overflow: hidden sería guardárselo todo. overflow: auto sería lo más sano: lo que no cabe tiene un canal de salida disponible — la barra de scroll aparece cuando hace falta. Las emociones que desbordan también necesitan un canal: no suprimirlas (hidden) ni explotarlas sin contexto (visible), sino encontrar el espacio adecuado para procesarlas (auto).

Hay diferencia entre ignorar a alguien y no verlo — y ambas duelen de formas distintas

Las habilidades relacionales incluyen "escuchar activamente" y "comunicarse claramente". Display: none es ignorar activamente — el grupo decidió no contar con esa persona. El overflow del salón: gestionar lo que desborda

Marisol tiene demasiado y no cabe en el espacio disponible. En CSS, overflow tiene cuatro respuestas posibles. En la vida también. Esta actividad propone que el grupo construya una página de "espacio de desahogo" — donde el overflow sea el tema central, tanto técnicamente como emocionalmente.

🧘 SEL · Autocontrol

overflow: auto es la estrategia más sana — el canal aparece cuando se necesita

El autocontrol incluye "manejar las emociones de manera efectiva". Overflow: hidden (guardar todo) y overflow: visible (dejarlo salir todo sin filtro) son los extremos que raramente funcionan bien. Overflow: auto es lo que los psicólogos llaman regulación emocional: el canal de salida existe y se activa cuando el contenido lo necesita, no antes. En tu bitácora:

  • → ¿Cuál de los cuatro overflow describe mejor tu propia estrategia para manejar lo que te desborda?

  • → ¿Qué "canal de salida" — persona, actividad, espacio — funciona como tu overflow: auto?
  • → ¿Cuál fue la metáfora de overflow que le costó más al grupo construir? ¿Por qué?

Criterios de valoración · Guía 13

¿Cómo se evaluará tu trabajo?

CriterioBásico (1–2)Satisfactorio (3)Superior (4–5)
Posicionamiento
Técnico		 Solo usa position: static o aplica mal absolute (sin padre posicionado). Usa fixed para el nav, relative + absolute para badges, y sticky en al menos un elemento. El posicionamiento es coherente y funcional. El z-index está pensado para la jerarquía visual deseada.
Display y visibilidad
Técnico		 Confunde display: none con visibility: hidden o no los usa. Usa display: none, inline-block y visibility: hidden correctamente en contextos apropiados. Cada uso de display y visibility está justificado en comentario CSS y se nota la diferencia técnica entre los valores.
Overflow manejado
Técnico		 No usa overflow o lo aplica sin sentido. Usa overflow: auto o hidden en al menos un contenedor con altura definida. El overflow está elegido con criterio — auto donde el contenido puede variar, hidden donde el diseño lo requiere.
Metáforas de vida
SEL		 No conecta las propiedades CSS con situaciones personales. Conecta position, display u overflow con al menos una situación real de su vida. Las metáforas son precisas y personales — overflow: auto no es "a veces guardas cosas" sino algo específico de su experiencia.
Reflexión sobre presencia e invisibilidad
SEL · Relacional		 No aborda la diferencia entre no estar y estar sin ser visto. Distingue entre display: none y visibility: hidden en términos relacionales. La reflexión muestra comprensión genuina de cómo se siente estar presente pero no percibido — y propone una acción concreta.
