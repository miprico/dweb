CSS Grid: construir estructuras que sostienen

CSS Grid es el sistema de diseño de dos dimensiones — columnas y filas al mismo tiempo. Aprenderás a crear cuadrículas flexibles, colocar elementos exactamente donde los necesitas y construir layouts de página completos. Y a través de eso, a reflexionar sobre las estructuras que sostienen: las que se pueden ver y las que trabajan en silencio.

Para empezar · Tres situaciones reales

Las estructuras que sostienen — visibles e invisibles

Lee estas tres situaciones antes de abrir el editor.

🏠En casa · La cuadrícula invisible de la familia

En la casa de los Ospina, todo funciona. Los desayunos están listos, los uniformes limpios, los recibos pagados, las citas médicas agendadas. Nadie tiene que coordinar nada porque la abuela Rosa tiene en la cabeza una cuadrícula completa: quién hace qué, cuándo, en qué espacio. Grid es exactamente eso — una cuadrícula que define qué va en cada celda, en qué columna, en qué fila. La abuela Rosa gestiona la casa con una lógica de grid que nadie ve pero todos aprovechan.

💭 ¿Hay alguien en tu familia que "gestiona el grid" — que tiene en la cabeza la estructura que hace funcionar todo?
📰En el colegio · El periódico mural

El comité del periódico mural tiene un problema: cada vez que alguien pega algo, el resultado se ve desordenado — textos encimados, fotos demasiado grandes, títulos perdidos. La presidenta del comité, Valentina, propone algo diferente: primero definan la cuadrícula — cuántas columnas, qué áreas son para títulos, cuáles para fotos, cuáles para texto. Primero la estructura, luego el contenido. Cuando aplicaron eso, el mural quedó diferente.

💭 ¿Hay algún proyecto o espacio en tu vida que necesita "primero definir la cuadrícula" antes de poner contenido?
🧩Con compañeros · Cada uno en su área

En el proyecto de artes, dividieron el lienzo en áreas: esquina superior izquierda para el título, banda central para la imagen principal, columna derecha para las notas, banda inferior para las firmas. Cada quien sabía exactamente qué área le correspondía. Nadie invadió el espacio de otro. El resultado fue coherente sin que nadie tuviera que controlarlo. Eso es grid-template-areas: nombrar cada zona y asignar quién va ahí.

💭 ¿En tu vida hay áreas bien definidas — zonas donde cada quien sabe qué le toca — y áreas donde todo se mezcla?

La abuela Rosa gestiona el hogar con una cuadrícula mental. Valentina propuso definir la estructura antes del contenido. El grupo de artes nombró cada área y respetó los límites. CSS Grid hace exactamente esto: define una cuadrícula precisa de filas y columnas, nombra las áreas y permite que cada elemento sepa exactamente dónde va.

01

display: grid — definir la cuadrícula

grid-template-columns · grid-template-rows · gap · fr · repeat()

Explicación

Con display: grid el contenedor se convierte en una cuadrícula. Los hijos directos se colocan automáticamente en las celdas. grid-template-columns define cuántas columnas hay y qué tan anchas son. grid-template-rows hace lo mismo para las filas. La diferencia clave con Flexbox: Grid trabaja en dos dimensiones a la vez — filas Y columnas simultáneamente.

Flexbox — una dimensión

Controla el eje principal. Ideal para filas o columnas de elementos.

Controla filas Y columnas. Ideal para layouts de página completos.

grid-base.css
.cuadricula { display: grid; /* grid-template-columns: define las columnas */ grid-template-columns: 200px 1fr 1fr; /* fija flexible flexible */ /* grid-template-columns: repeat(3, 1fr); 3 columnas iguales */ /* grid-template-columns: repeat(3, 200px); 3 columnas de 200px */ /* grid-template-rows: define las filas */ grid-template-rows: 80px auto 60px; /* header contenido footer */ /* gap: espacio entre celdas */ gap: 20px; /* igual entre filas y columnas */ /* gap: 16px 24px; row-gap column-gap */ } /* fr = fracción del espacio disponible */ /* 1fr 2fr 1fr → la del medio ocupa el doble que las otras */ .con-fr { grid-template-columns: 1fr 2fr 1fr; } /* repeat(n, valor): forma corta de repetir */ .galeria { display: grid; grid-template-columns: repeat(4, 1fr); /* 4 columnas iguales */ gap: 16px; } /* minmax(mín, máx): columna que se adapta entre dos valores */ .responsiva { grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); /* crea tantas columnas como quepan de mínimo 250px */ }

auto-fit vs auto-fill: repeat(auto-fit, minmax(250px, 1fr)) crea tantas columnas como quepan de mínimo 250px — y si sobra espacio, las columnas crecen. auto-fill hace lo mismo pero deja columnas vacías si hay espacio sobrante. auto-fit es el más usado en galerías responsivas — funciona en cualquier pantalla sin media queries.

02

Ubicar elementos: grid-column, grid-row y grid-area

Decirle a cada elemento exactamente dónde va — y hacer que algunos ocupen más de una celda

Explicación

Por defecto, los elementos de un grid se colocan automáticamente, uno por celda, de izquierda a derecha y de arriba hacia abajo. Pero con grid-column y grid-row puedes decirle a un elemento que ocupe varias celdas — que "se extienda" a lo largo de columnas o filas. La notación usa las líneas de la cuadrícula, que se numeran empezando desde 1.

Las líneas de la cuadrícula — empiezan desde 1:

grid-posicion.css
.cuadricula { display: grid; grid-template-columns: repeat(3, 1fr); grid-template-rows: 80px 1fr 60px; gap: 16px; } /* grid-column: línea-inicio / línea-fin */ .header { grid-column: 1 / 4; /* de línea 1 a línea 4 = toda la fila */ /* grid-column: 1 / -1; -1 = última línea (más fácil) */ /* grid-column: span 3; ocupa 3 columnas desde donde está */ } .sidebar { grid-column: 1 / 2; grid-row: 2 / 3; } .main { grid-column: 2 / 4; grid-row: 2 / 3; } .footer { grid-column: 1 / -1; /* 1 / -1 siempre cubre todo el ancho */ grid-row: 3 / 4; }

03

grid-template-areas: nombrar las zonas

La forma más legible de definir un layout — como el mural de Valentina

Explicación

grid-template-areas permite nombrar las zonas de la cuadrícula usando palabras. Es la forma más legible de definir un layout complejo porque el código se parece visualmente a la distribución real de la página. Como el comité del periódico mural de Valentina — primero se nombran las áreas, luego cada elemento dice a qué área pertenece.

grid-areas.css
.pagina { display: grid; grid-template-columns: 240px 1fr; grid-template-rows: 70px 1fr 60px; gap: 16px; min-height: 100vh; /* Las comillas definen cada fila. Los nombres, cada celda */ /* Un punto ( . ) = celda vacía */ grid-template-areas: "header header" "sidebar main " "footer footer"; } /* Cada elemento declara su área con grid-area */ header { grid-area: header; } aside { grid-area: sidebar; } main { grid-area: main; } footer { grid-area: footer; } /* place-items: center — centra todos los items */ .centrado { display: grid; place-items: center; /* align-items + justify-items en uno */ } /* Ejemplo con celda vacía */ .con-vacio { grid-template-areas: "header header header" "sidebar main . " /* punto = celda vacía */ "footer footer footer"; }
Así se ve un grid con template-areas — el mural de Valentina:

Grid vs Flexbox — cuándo usar cada uno: Usa Flexbox para distribuir elementos en una dirección (nav, lista de botones, tarjetas en fila). Usa Grid para layouts completos de página — cuando necesitas controlar filas Y columnas al mismo tiempo. En la práctica, se usan juntos: Grid para la estructura de la página, Flexbox para los componentes dentro de esa estructura.

El layout de mi proyecto con Grid

🗺️Como Valentina — primero la cuadrícula, luego el contenido

Valentina propuso que antes de pegar cualquier cosa, definieran la estructura. Hoy harás lo mismo con tu proyecto: antes de escribir contenido, dibuja en papel el grid de tu página. ¿Cuántas columnas? ¿Qué áreas tiene? ¿Dónde va cada sección? Esa arquitectura invisible es lo que hace que todo lo demás funcione.

💭 ¿Cuál es la "cuadrícula" de tu vida — la estructura invisible que organiza tus días y tus roles?
  • 1

    Dibuja en papel el grid de tu index.html

    Antes de tocar el código: dibuja la cuadrícula de tu página. ¿Cuántas columnas? ¿Cuántas filas? ¿Qué nombre tiene cada área? El dibujo en papel es el grid-template-areas antes de escribirlo en código.

  • 2

    Implementa el layout con grid-template-areas

    Convierte el layout de tu página principal en un grid con grid-template-areas. Mínimo: header, nav, main, aside y footer. Usa grid-area en cada elemento HTML para asignarlo a su área nombrada.

  • 3

    Crea una galería responsiva con auto-fit y minmax

    En mi_historia.html, la sección de intereses o habilidades como galería de tarjetas: grid-template-columns: repeat(auto-fit, minmax(220px, 1fr)). Sin media queries — funciona en cualquier pantalla.

  • 4

    Usa grid-column: span para un elemento destacado

    En la galería, hay una tarjeta que es más importante que las demás. Dale grid-column: span 2 para que ocupe dos columnas. ¿Qué elemento de tu historia merece ese espacio extra?

📋 Tu CSS debe incluir
  • Layout principal con display: grid + grid-template-areas en index.html
  • grid-area en header, nav, main, aside y footer
  • Galería responsiva con repeat(auto-fit, minmax(220px, 1fr))
  • Al menos un elemento con grid-column: span 2 justificado en comentario CSS
  • Uso de la unidad fr en la definición de columnas
  • Comentarios CSS que expliquen las decisiones de diseño del grid

A2

Dos grids para la misma historia: distintas estructuras, distinto énfasis

Parejas · 20 minutos · Toma de decisiones — cómo la estructura cambia lo que se percibe primero
⏱ 20 min 👥 Parejas 💻 Mismo HTML, dos grids ⚖️ SEL: Toma de Decisiones
💡Volvamos al periódico mural — ¿qué va en el área grande?

En el periódico mural, el área más grande decide qué es lo más importante. Si la imagen ocupa 2fr y el texto solo 1fr, la imagen domina. Si el texto ocupa 3fr y la imagen solo 1fr, el mensaje domina. La distribución de la cuadrícula es una decisión editorial — dice qué importa más.

💭 En tu vida, ¿qué área ocupa más espacio en tu "cuadrícula" cotidiana? ¿Es el área que quisieras que ocupara más?
  • 1

    Usen el mismo HTML — una página con 4 secciones de contenido personal

    Header con nombre, sección "mis logros", sección "mis dificultades", sección "mis sueños" y footer. Igual en los dos archivos.

  • 2

    Cada uno diseña un grid diferente que cambia el énfasis (8 min)

    Persona A: "mis logros" ocupa 3fr, las demás 1fr. La estructura dice que los logros son lo más importante. Persona B: "mis sueños" ocupa 3fr, las demás 1fr. La estructura dice que el futuro es lo más importante. Implementen como grid-template-areas y fr distintas.

  • 3

    Comparen y conversen (5 min)

    ¿La estructura cambia lo que el lector percibe como más importante? ¿Cuál versión los representa mejor? ¿Hay una versión que les incomoda y por qué?

📋 Producto esperado A2
  • Dos versiones CSS con grid-template-areas y fr distintos para el mismo HTML
  • Comentario CSS explicando la "decisión editorial" de cada versión
  • Un párrafo reflexivo: ¿cuál versión los representa mejor y por qué?
⚖️ SEL · Toma de Decisiones

Decidir la estructura es decidir qué importa más — en la página y en la vida

La toma de decisiones responsables incluye "anticipar y evaluar las consecuencias". La distribución del grid determina lo que el ojo ve primero, lo que ocupa más espacio, lo que parece más importante. Esas decisiones tienen consecuencias en cómo el lector percibe tu historia. En tu bitácora:

  • → ¿La versión que diseñaste dice algo de cómo priorizas tu propia historia?
  • → ¿Hay algo que en tu vida actual ocupa más espacio en tu "grid" del que quisieras, y algo que merece más fr del que tiene?
  • → ¿Cuál de las dos versiones te incomodó más ver? ¿Por qué?
A3

El mural del salón: un grid colectivo con áreas nombradas

Grupos de 4 · 20 minutos · Conciencia social — definir juntos qué va en cada área
⏱ 20 min 👥 Grupos de 4 💻 Archivo colectivo 🌍 SEL: Conciencia Social
📰Volvamos al comité de Valentina — primero la cuadrícula

Valentina propuso primero nombrar las áreas del mural. Esta actividad hace lo mismo: el grupo diseña un "mural digital" con grid-template-areas donde cada integrante tiene su área nombrada — y la distribución de fr refleja cómo el grupo valora cada parte colectivamente.

💭 Si el grupo fuera un grid, ¿qué área ocuparía más espacio? ¿Qué área nunca tiene espacio?
  • 1

    Diseñen el grid del mural en papel primero (5 min)

    ¿Cuántas columnas? ¿Cuántas filas? ¿Qué áreas tiene el mural? ¿Cuánto fr merece cada área? Dibujo primero, código después.

  • 2

    Construyan el archivo mural_salon.html (10 min)

    Con un grid que use grid-template-areas con nombres que representen lo que el grupo quiere mostrar. Un área por integrante más áreas colectivas (logros, sueños, algo que los une). Cada área con su color y contenido real.

  • 3

    Consensúen el fr de cada área (5 min)

    ¿Tiene cada integrante el mismo fr? ¿Hay áreas colectivas que merecen más espacio? La decisión debe ser por consenso y documentada en un comentario CSS.

📋 Producto esperado A3
  • Archivo mural_salon.html con grid-template-areas que incluya a todos
  • Cada integrante con su área nombrada y un elemento HTML con grid-area
  • Áreas colectivas con fr consensuado y documentado
  • Un footer colectivo con reflexión: ¿el grid refleja cómo funciona el grupo realmente?
🌍 SEL · Conciencia Social

La estructura que diseñamos juntos refleja cómo entendemos el grupo

La conciencia social incluye "reconocer las fortalezas de los demás". El grid-template-areas que diseñaron dice qué áreas nombran y cuánto espacio le dan a cada quien. ¿Hay alguien cuya área quedó con poco fr? ¿Hay áreas colectivas que nadie quiso nominar? En tu bitácora:

  • → ¿Fue fácil nombrar las áreas del mural? ¿Hubo algún área que nadie quería reclamar?
  • → ¿Todos los integrantes tienen el mismo fr en el grid o hay diferencias? ¿Por qué?
  • → ¿El mural resultante representa al grupo como realmente es, o como quisieran que los vieran?
  • → ¿Hay algo de la "estructura invisible" de tu grupo — como la de la abuela Rosa — que el grid no logró capturar?

Criterios de valoración · Guía 15

¿Cómo se evaluará tu trabajo?

CriterioBásico (1–2)Satisfactorio (3)Superior (4–5)
Grid funcional con fr
Técnico		 Usa display: grid pero solo con píxeles fijos — sin fr ni repeat(). Usa fr y repeat() correctamente. El grid se adapta al contenido disponible. La combinación de fr y píxeles/minmax refleja una decisión consciente sobre qué elementos son flexibles y cuáles tienen tamaño fijo.
grid-template-areas
Técnico		 No usa grid-template-areas o los nombres no coinciden con grid-area en los items. Usa grid-template-areas correctamente con al menos 4 áreas nombradas. El grid-template-areas se lee como el layout visual de la página — alguien que lo lea en el CSS puede imaginar la página sin verla.
Galería responsiva
Técnico		 La galería tiene número fijo de columnas — se rompe en pantallas pequeñas. Usa repeat(auto-fit, minmax()) para galería responsiva sin media queries. La galería funciona perfectamente en móvil y escritorio, y al menos un elemento tiene grid-column: span justificado.
Estructura como decisión
SEL · Autoconciencia		 El grid es técnicamente correcto pero sin reflexión sobre por qué esa estructura. Los comentarios CSS explican por qué cada área tiene el tamaño que tiene. La estructura del grid refleja claramente qué considera más importante el estudiante — la distribución de fr tiene una lógica personal explícita.
Cuadrícula colectiva
SEL · Conciencia Social		 El mural del grupo no refleja a todos los integrantes. Todos los integrantes tienen área nombrada y el fr fue acordado. La reflexión colectiva en el footer del mural es honesta sobre si el grid refleja la dinámica real del grupo — sin maquillar.
